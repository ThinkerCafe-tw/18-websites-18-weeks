# 第三講　攝影師作品集　作業

> 格式參照前學年函授作業（逐講、選擇＋是非＋簡述＋實作；題目在前、解答在後）。內容對應 115 上《網頁設計》第三講。

## 壹、作業題目

### 一、選擇題（單選）

1. （　）一位準新娘在 Google 搜尋「台中婚紗攝影」後點進攝影師的作品集網站，相較於只刷攝影師的 Instagram，網站最能幫她做到下列何者？
   (1) 看到攝影師每天的限時動態與生活近況
   (2) 在短時間內看懂風格、找到分類作品與方案價格，快速判斷適不適合自己
   (3) 即時私訊攝影師聊天
   (4) 累積按讚數與粉絲數

2. （　）本講要求把攝影師作品集從「電腦裡的 `index.html`」推進成「正式網站」，下列哪一條才是本講教的完整流程？
   (1) AI 生成 → 直接把檔案寄給客人看
   (2) AI 生成 → 本機檢查 → GitHub 保存 → Vercel 部署 → 手機驗收 → 持續更新
   (3) AI 生成 → 印出來貼在工作室牆上
   (4) AI 生成 → 上傳到 Instagram 限時動態

3. （　）關於攝影師作品集的「首頁第一屏」，下列敘述何者**正確**？
   (1) 第一屏應放滿兩百張作品，讓客人一次看個夠
   (2) 「用光影記錄生命中最美好的瞬間」這種句子比有地點與服務類型的說明更實用
   (3) 第一屏應同時回答「你拍什麼類型、你的風格是什麼、我可以怎麼聯絡或看方案」三件事
   (4) 為了乾淨，首頁不需要放任何按鈕

4. （　）部署上線後，作品集「本機打開正常、線上卻看不到照片」最常見的原因是下列何者？
   (1) 客人的網路太慢
   (2) 圖片路徑錯、檔名大小寫不一致，或圖片檔根本沒上傳到 repository
   (3) Vercel 一定要付費才會顯示圖片
   (4) Instagram 沒有同步

### 二、是非題

5. （　）AI 生成攝影師網站時自動補上「服務過上百組新人」「獲得多項國際獎項」「客戶一致好評」等文字，只要看起來專業，就算不是事實也可以直接保留上線。

6. （　）`commit message` 只是寫給工程師看的技術術語，攝影師本人完全用不到，所以隨便寫什麼都沒差。

### 三、簡述題

7. 本講提到攝影師作品集網站要解決的五件事是什麼？請簡述每一項的意思。

8. 請用自己的話說明 GitHub 與 Vercel 在本講的「角色分工」，並說明為什麼修改一段首頁文案、commit 之後要回頭觀察 Vercel 有沒有自動重新部署。

### 四、實作題

9. 請用本講的提示詞讓 AI 工具產出一個攝影師作品集 `index.html`，至少放入三個作品分類（每類至少四張示意圖），建立 GitHub repository 完成第一次 commit，並用 Vercel 部署取得公開網址。接著用手機打開網址，記錄首頁、作品集、詢問表單三個畫面，列出至少兩個需要修改的地方與你打算用的修改句；最後修改一段首頁文案、commit 後確認 Vercel 自動更新。

## 貳、作業解答

### 一、選擇題

1. **（2）**
   解析：攝影師的 Instagram 像一條一直往下流的河，作品會被新貼文蓋掉、分類不清楚、價格也找不到。作品集網站則像一間整理好的工作室，能讓客人在短時間內看懂風格、進入自己想要的作品分類、看清楚方案與價格，五分鐘內判斷這位攝影師適不適合自己。(1)(3)(4) 是社群平台的特性，正好是本講要用網站補起來的缺口。

2. **（2）**
   解析：本講的核心是「版本控制與線上部署」，要讓 AI 產出的網頁從一個檔案變成可經營的網站。完整生命線是 AI 生成 → 本機檢查 → GitHub 保存 → Vercel 部署 → 手機驗收 → 持續更新。(1)(3)(4) 都停在「檔案」階段，沒有保存、發布與更新流程，網站一旦要改照片、改價格就會變成混亂的人工搬運。

3. **（3）**
   解析：攝影師網站的首頁第一屏應同時回答三件事——你拍什麼類型、你的風格是什麼、我可以怎麼聯絡或看方案。(1) 第一版作品先放 12 到 24 張代表作即可，訪客不是評審，不會看兩百張；(2) 空泛的抒情句不如「台中婚紗與婚禮紀錄攝影，替你留下自然、不過度修飾的真實情緒」這種有地點、服務類型與風格判斷的句子；(4) 首頁需要「看作品」「詢問檔期」這類明確的下一步按鈕。

4. **（2）**
   解析：圖片路徑錯誤是作品集部署後最常見的失敗原因之一。HTML 寫 `images/wedding-01.jpg`，GitHub repository 裡就必須真的有同一個路徑、同樣大小寫的檔案。網站本機開起來正常，不代表上線後正常，大小寫、資料夾名稱、副檔名都要一致。(1)(3)(4) 都不是這個現象的真正原因。

### 二、是非題

5. **（×）**
   解析：這些是 AI 很容易自動補上的「假作品、假客戶、假評價」。如果不是事實就必須刪掉。攝影師的作品集最怕信任破口——一旦訪客覺得文字在包裝，連真實的照片也會一起變得不可信。建立信任要靠具體、可驗證的內容（真實流程、透明價格、明確交件），不是靠編造的獎項與好評。

6. **（×）**
   解析：commit message 不只是給工程師看的。它記錄「更新首頁照片」「修改方案價格」「調整手機版表單」等每一次有說明的存檔。三個月後攝影師要回想「我什麼時候改過價格」，commit 紀錄就是答案。所以每次修改都要讓 commit message 說清楚你改了什麼，這是攝影師自己日後維護網站的依據。

### 三、簡述題

7. **答**：攝影師作品集網站至少要解決五件事——
   (一) **讓客人快速看懂你的風格**：社群貼文是時間流，客人不想考古；網站要讓他短時間內知道你最擅長什麼、風格是否穩定。
   (二) **讓作品可以被分類**：婚紗、人像、商業、活動紀錄背後的客人、決策與預算都不同，要分開呈現，讓客人直接進入自己的情境。
   (三) **讓搜尋引擎看得懂**：用標題、描述、區塊結構與文字，清楚告訴 Google 你是誰、在哪裡、提供什麼攝影服務、適合哪種客人。
   (四) **讓信任資訊集中**：流程是否清楚、費用是否透明、交件是否明確、要跟誰合作，這些都集中在網站上回答。
   (五) **讓網站可以持續維護**：攝影師會不斷有新作品、新方案、新價格，網站不能「建好就死掉」，要能持續更新。

8. **答**：GitHub 與 Vercel 的角色分工是——
   - **GitHub**：保存網站檔案與每一次有說明的修改紀錄（commit），它是「保存新版本」的地方。
   - **Vercel**：讀到 GitHub 上的新版本後，自動建立一個新的 deployment，把網站發布成公開網址，它是「把版本變成上線網站」的地方。
   為什麼改完文案、commit 後要回頭看 Vercel 有沒有自動重新部署：因為這一步能親眼驗證兩者的分工確實接上了——GitHub 收到新 commit，Vercel 偵測到 `main` branch 更新並自動產生新部署，正式網址才會顯示新版本。若 Vercel 沒更新，通常是 commit 沒成功、或部署還在進行中，這時就要回去看 GitHub 的 commit 與 Vercel 的 deployments 狀態。

### 四、實作題

9. **答（參考評分重點）**：本題為實作，無單一標準答案，評分看四件事——
   (1) **是否完成整條上線流程**：用提示詞產出含三個分類（每類至少四張示意圖）的 `index.html`、建立 GitHub repository 並完成第一次 commit、用 Vercel 部署取得公開網址、再用手機打開驗收首頁／作品集／詢問表單三個畫面。
   (2) **記錄的問題是否具體**：例如「線上圖片不見，HTML 寫 `images/Wedding-01.jpg` 但 repo 裡是 `wedding-01.jpg`，大小寫不一致」「手機版作品集沒變單欄，照片擠成一團」「詢問表單在手機上按鈕太小不好點」屬具體；「不好看」「怪怪的」過於籠統。
   (3) **修改句是否可操作**：例如「請補上手機版 RWD 規則，桌機作品集三欄、平板兩欄、手機單欄，表單欄位手機改單欄、按鈕高度至少 44px」「請把 HTML 的 `<title>` 與 `meta description` 加入攝影師名稱、服務類型與地區」「請刪掉自動產生的得獎與客戶好評文字」。並確認修改後其他區塊沒有被弄壞。
   (4) **是否驗證了更新流程**：改一段首頁文案、commit 後，是否確實回頭觀察到 Vercel 自動產生新 deployment、正式網址顯示新版本。能說出「GitHub 保存、Vercel 部署」這條分工，比只是把流程跑完更能展現理解。
