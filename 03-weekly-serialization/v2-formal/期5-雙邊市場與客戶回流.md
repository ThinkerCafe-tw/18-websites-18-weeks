## 壹、課業輔導

### 一、本期學習定位

本期涵蓋教材第十三至第十五講，三者處理之核心問題為「客戶關係的長期經營」。家教媒合之核心為「供需匹配」，屬雙邊市場設計；汽車保養之核心為「定期回流」，屬會員生命週期管理；才藝教室之核心為「招生漏斗」，屬行銷轉換設計。本期補充三項教材未展開之重點：雙邊市場之冷啟動策略、客戶回流之生命週期模型、以及招生漏斗之關鍵節點設計。

### 二、重點整理與補充

#### （一）雙邊市場之冷啟動策略

家教媒合網站（第十三講）為典型雙邊市場：一邊為家教老師（供給方），另一邊為學生家長（需求方）。雙邊市場之共同難題為「冷啟動」——沒有學生家長，老師不願註冊；沒有老師，學生家長不會使用。學界稱此為「雞生蛋問題」，業界有四種常見解法。第一為「單邊先行」：Airbnb 初期先手動招募房東，待累積 300 間以上始開放給房客；應用於家教平臺為先主動邀請現役老師註冊，並提供免費曝光期。第二為「自給自足」：平臺方自行扮演其中一邊，例如 Google Maps 初期由 Google 自行建置地圖，再開放使用者貢獻。第三為「模擬需求」：以優惠或補貼吸引需求方先進場，讓供給方看到流量後自主加入；應用於家教平臺為推出「新生首月半價」活動。第四為「垂直切入」：不同時覆蓋全領域，先切入單一領域（例如僅收小學數學家教），建立密集度後再擴張。教材第十三講採第四種策略作為範例。冷啟動之核心原則為：不追求即時雙邊平衡，而追求單邊之足夠密集度，讓另一邊自然加入。

#### （二）客戶回流之生命週期模型

汽車保養業（第十四講）之回流週期約為 5,000 公里或半年，屬典型之「可預測性回流」產業。類似產業包含牙醫（半年洗牙）、眼鏡行（一年配鏡）、美容院（一至兩個月剪髮）等。此類業者之客戶生命週期可分為四階段。第一階段為「新客」，首次接觸至第三次回流前，此階段應重於服務體驗與聯絡方式取得。第二階段為「熟客」，累積三至十次回流，此階段應重於個人化服務與偏好建檔。第三階段為「老客」，持續往來超過一年，此階段應重於情感連結與口碑引薦。第四階段為「流失客」，超過預期週期一倍仍未回流者，此階段應重於原因診斷與重啟誘因。各階段之觸發訊息應差異化設計：新客收到「首次保養後 3 個月之保養貼士」，熟客收到「距離上次保養已 5 個月，建議預約」，老客收到「感謝您這一年的信任 + 年度 VIP 回饋」，流失客收到「我們發現您已 9 個月未來，是否有我們可改進之處」。教材第十四講之回流提醒採第二階段邏輯，本期擴充至全生命週期管理。

#### （三）招生漏斗之關鍵節點

才藝教室（第十五講）之招生流程屬典型「行銷漏斗」（marketing funnel）。漏斗由上至下可分為五層：曝光（Awareness）、興趣（Interest）、考慮（Consideration）、試課（Trial）、報名（Enrollment）。每一層皆有流失，設計網站時應針對每層之流失原因提供對策。曝光層之流失通常因「廣告受眾不精準」，對策為以 Google Analytics 分析流量來源並調整廣告投放。興趣層之流失通常因「首頁未於三秒內傳達教室特色」，對策為首頁 hero 區明確呈現差異化（例如「全臺第一家 3 至 6 歲兒童桌球教室」）。考慮層之流失通常因「費用與時間不透明」，對策為明確標示每期學費、課程長度、教師資歷。試課層之流失通常因「預約試課門檻過高」，對策為提供「免費試聽一堂」且預約流程不超過三步驟。報名層之流失通常因「付款方式受限」，對策為提供信用卡、LINE Pay、分期付款等多元選項。監測方式為於各層設定轉換事件並追蹤流失率，若某層遠高於業界平均（各層保留約 30–50%），即為需優化之訊號。

### 三、實例分析：汽車保養業從紙本里程表到智慧提醒之四階段

**第一階段為「紙本里程表」**。老闆於每次保養時於客戶之保養手冊上手寫里程數與日期。客戶自行保管，下次回來時攜帶翻閱。此階段無系統性回流機制，完全依賴客戶主動記得。回流率業界觀察約為 30–40%。**第二階段為「LINE 群組提醒」**。老闆將所有客戶加入 LINE，於每月發送「本月優惠」訊息。此階段客戶接收率提升，但訊息為群發形式，無法個人化，長期易被視為騷擾而封鎖。回流率約提升至 45–55%。**第三階段為「試算表手動提醒」**。老闆以 Excel 或 Google Sheets 建立客戶資料表，欄位包含姓名、車款、上次保養日期、下次建議保養日期；每月篩選「30 天內需保養」之客戶，以個人化訊息主動聯繫。回流率約提升至 60–70%。**第四階段為「自動化提醒系統」**。建立會員網站（教材第十四講），系統依「上次保養日期 + 建議間隔」自動觸發個人化提醒。業者得以脫離每月手動操作之負擔，回流率可穩定於 75% 以上，且有餘裕服務更多客戶。教材第十四講所建置者即此階段之實作。

## 貳、習題與解答

### 第一題【選擇題】

雙邊市場之「冷啟動」問題中，下列何種策略屬於「單邊先行」？

(A) 平臺以優惠吸引供需兩方同時進場
(B) Airbnb 初期手動招募房東，累積 300 間以上始開放給房客
(C) 政府補貼平臺一年運作費用
(D) 讓使用者自行建立平臺內容

**答：(B)**

**解析**：「單邊先行」意指平臺於初期集中資源於一側，讓該側達到足夠密集度後，另一側自然加入。(B) 為經典案例。(A) 為雙邊同時操作非單邊先行。(C)(D) 屬不同策略類型。

### 第二題【選擇題】

汽車保養業之客戶生命週期中，「流失客」階段應優先採取何種行動？

(A) 推薦朋友享優惠
(B) 原因診斷與重啟誘因
(C) 提供 VIP 升級
(D) 發送最新服務優惠訊息

**答：(B)**

**解析**：流失客已超過預期回流週期一倍仍未回流，單純推送優惠訊息常無效。正確行動為先診斷原因（是否已換車？是否有不滿意之處？），再據以提供對應之重啟誘因。若未診斷即發送優惠，有時反而加深客戶對業者「只顧銷售不在乎我」之印象。

### 第三題【選擇題】

才藝教室招生漏斗之「試課層」流失率偏高，**最可能**之原因為？

(A) 廣告受眾不精準
(B) 首頁視覺不夠精美
(C) 預約試課門檻過高
(D) 教室地點偏遠

**答：(C)**

**解析**：試課層之特徵為「使用者已對教室產生興趣並評估中」，此時若預約流程繁瑣（需填寫過多欄位、需下載 App、需實體到訪領表），將造成大量流失。(A) 為曝光層問題。(B) 為興趣層問題。(D) 於試課階段已被評估過，非試課層之主要流失原因。

### 第四題【是非題】

雙邊市場冷啟動時，應同時對供需雙方投入等量資源以維持平衡。（　）

**答：（×）**

**解析**：冷啟動之核心原則為「優先追求單邊密集度，讓另一邊自然加入」，而非同時平衡。若資源分散於兩側，常導致兩側皆未達到有效密集度而雙雙失敗。Airbnb、Uber、Amazon 等成功之雙邊市場，無一於初期採「等量投入」策略。

### 第五題【是非題】

汽車保養業之 LINE 群組發送每月優惠訊息，是建立客戶回流機制之最佳方式。（　）

**答：（×）**

**解析**：LINE 群組發送為「第二階段」之作法，回流率僅 45–55%，且訊息無法個人化，長期易被封鎖。較佳作法為「第三階段」（試算表手動提醒）或「第四階段」（自動化提醒系統），依每位客戶之上次保養日期與車款個人化發送訊息，回流率可達 70% 以上。

### 第六題【應用題】

試設計一組 Google Sheet 公式，用以自動標示「最近 30 天內需保養」之客戶。假設 A 欄為客戶姓名、B 欄為上次保養日期、C 欄為建議保養間隔天數（通常 180），請於 D 欄標示「需提醒」。

**答**：

D2 格之公式：

```
=IF(
  AND(
    TODAY() - B2 >= C2 - 30,
    TODAY() - B2 <= C2
  ),
  "需提醒",
  ""
)
```

**解析**：此公式判斷距離上次保養之天數是否落於「建議間隔前 30 天」至「建議間隔當日」之區間。若是則顯示「需提醒」，否則留空。將公式向下拖曳即可套用於所有客戶。進階作法可再加條件格式（Conditional Formatting），使「需提醒」之列顯示黃色背景，並透過「篩選」功能快速匯出當月需提醒之客戶名單。此為教材第十四講之試算表版簡化實作，適合客戶數 50–200 之中小型業者。

### 第七題【應用題】

試以 JavaScript 撰寫一段函式，依客戶之上次保養日期與保養間隔，判斷客戶目前處於「新客／熟客／老客／流失客」之哪一階段。

**答**：

```javascript
function classifyCustomer(customer) {
  const now = new Date();
  const lastServiceDate = new Date(customer.lastServiceDate);
  const daysSinceLastService = Math.floor(
    (now - lastServiceDate) / (1000 * 60 * 60 * 24)
  );
  const expectedInterval = customer.expectedIntervalDays || 180;

  // 流失客：超過預期週期一倍
  if (daysSinceLastService > expectedInterval * 2) {
    return 'lost';
  }

  // 依回流次數分類
  const visitCount = customer.totalVisits;

  if (visitCount >= 10) {
    return 'vip';           // 老客
  } else if (visitCount >= 3) {
    return 'regular';       // 熟客
  } else {
    return 'new';           // 新客
  }
}

// 使用範例
const customer = {
  name: '王小姐',
  lastServiceDate: '2025-08-15',
  totalVisits: 5,
  expectedIntervalDays: 180
};

console.log(classifyCustomer(customer));  // 輸出: 'regular'
```

**解析**：此函式之設計重點有二。第一，**先判斷流失客**再判斷其他階段，因為流失客可能本為老客但已久未回流，階段判斷應以「時間」優先於「次數」。第二，**參數化建議間隔**（`expectedIntervalDays`），因不同服務之間隔不同（保養 180 天、洗牙 180 天、剪髮 45 天），參數化設計可同函式用於多種產業。實務應用時，各階段應搭配不同之行銷訊息範本。

### 第八題【情境題】

某才藝教室首頁訪客每日約 300 人，但試課預約每日僅 2 人。請診斷漏斗何層流失最嚴重，並提出三項可驗證的改善假設。

**答**：

**初步診斷**：

需先收集各層數據（以 Google Analytics 或類似工具）。假設數據如下：

- 首頁訪客：300 人
- 點擊「課程介紹」：90 人（興趣層保留 30%）
- 點擊「費用與時間」：60 人（考慮層保留 67%）
- 點擊「預約試課」：10 人（試課層保留 17%）
- 完成試課預約：2 人（報名層保留 20%）

由此可見，**試課層保留率（17%）遠低於其他層**，為流失最嚴重之節點。此可對應之原因為「考慮後有意試課，但於預約流程放棄」。

**三項可驗證之改善假設**：

1. **假設一：預約表單欄位過多導致放棄**。驗證方式：將必填欄位由 10 項減為 4 項（姓名、手機、欲試課日期、孩子年齡），觀察試課預約率是否提升。
2. **假設二：試課時段選擇不夠彈性**。驗證方式：由「僅週三下午 3:00」擴充為「週三、週六多時段」，觀察預約率是否提升。
3. **假設三：費用不透明導致卻步**。驗證方式：於預約頁面明確標示「首次試課免費、正式課程每期 NT$4,800」，觀察預約率是否提升。

每項假設建議以 2 週為測試週期，單一變數修改以避免混淆。

### 第九題【情境題】

某家教媒合平台剛開始運作，目前僅有 15 位老師註冊、30 位家長瀏覽。請提出一份冷啟動計畫，使平台於三個月內達到可持續經營之規模。

**答**：

**第一個月：垂直切入 + 單邊先行**

- 選定單一領域：例如僅限「台北市小學生數學家教」
- 主動招募老師：由創辦人至師範大學、補教業者徵求兼職老師，目標 30 位
- 提供老師首三個月 0 抽成以降低進入門檻
- 平台不開放家長註冊，先建立密集供給

**第二個月：家長方冷啟動**

- 同樣以垂直領域為廣告受眾（台北市小學生家長）
- 提供「首次配對 NT$500 折扣」誘因
- 目標：配對成功案例 20 組
- 建立案例頁並徵求家長具名評價

**第三個月：擴大範圍**

- 家教領域擴大至國中數學、英文、自然
- 地區擴大至新北市
- 依第二個月之家長反饋調整平台介面與配對流程
- 建立「家長評價」機制以建立信任資產

**關鍵指標監測**：

- 老師活躍率（本月接案之老師佔總註冊之比例）
- 家長轉換率（瀏覽至實際配對之比例）
- 配對滿意度（完成一期課程後之評分）

若三個月後老師活躍率低於 30% 或配對滿意度低於 4 星（5 星制），應暫停擴張並重新檢視供需匹配邏輯。

### 第十題【問題與討論】

試選擇一項你熟悉之產業（例如早餐店、便當店、補習班、家庭代工等），分析其是否具備「可預測性回流」特徵。若具備，請設計一份客戶生命週期管理計畫；若不具備，請說明原因並提出替代之客戶關係經營策略。

**答（參考架構）**：

本題無標準答案，評分重點在於：

1. **是否正確判斷產業之回流特徵**（可預測性回流 vs 不可預測性回流 vs 單次性消費）；
2. **若為可預測性回流**：是否能設計符合該產業週期之提醒機制（保養 180 天、剪髮 45 天、便當餐 7 天）；
3. **若為不可預測性回流**：是否能提出替代策略（例如：以季節性商品觸發回流、以社群經營維持品牌記憶）；
4. **是否考量客戶之主動性與被動性差異**。

參考範例（補習班）：
- 特徵：屬「週期性強制回流」（每週固定上課），客戶生命週期管理重點非在「觸發回流」而在「續報率」。
- 計畫：建立「學期結束前 4 週」「學期結束前 2 週」「學期結束前 1 週」三階段續報提醒，並差異化設計：第一階段以成績進步展示為主，第二階段以同儕續報情形為主，第三階段以續報優惠為主。

參考範例（便當店）：
- 特徵：介於可預測性回流與不可預測性之間，屬「日用品類消費」。
- 計畫：經營重點非個別客戶提醒，而在「每日新菜資訊觸達」與「集點卡實體接觸」。LINE 官方帳號發送當日菜單，配合實體集點卡（滿 10 次折 50 元）建立習慣。

評分不看結論，看是否能依產業特性提出合理對應策略。
