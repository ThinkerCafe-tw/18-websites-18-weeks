## 壹、課業輔導

### 一、本期學習定位

本期涵蓋教材第一至第三講，是整學期最重要的地基。第一講建立「學員與 AI 協作開發」的工作模式，第二講將設計思維從功能提升至美學層次，第三講則以攝影師作品集為題，處理「內容排序與敘事」此一作品集型網站的核心問題。本期課業輔導補充三項教材因篇幅限制而未深入之重點：HTML 語義化標籤的選用原則、CSS 排版的 Flexbox 與 Grid 時機判斷、以及作品集網站的資訊架構設計。

### 二、重點整理與補充

#### （一）HTML 語義化標籤的選用原則

教材第一講建立了 HTML 基本結構，但對語義化標籤的選用時機未完整展開。語義化標籤之目的有三：一為搜尋引擎爬蟲之可讀性，二為無障礙輔助科技（螢幕閱讀器）之辨識，三為團隊協作之可維護性。常見誤用有二：第一，將所有區塊皆以 `<div>` 包覆，喪失語義資訊；第二，誤將 `<section>` 與 `<article>` 混用。選用原則為：頁首導覽採 `<header>`，主內容採 `<main>`，獨立可轉貼之內容採 `<article>`（如部落格單篇、作品集單件），主題性區塊採 `<section>`（如「關於我們」「最新消息」），側邊欄採 `<aside>`，頁尾採 `<footer>`。以咖啡廳網站為例，店家簡介應以 `<section>` 包覆，而每一則菜單項目則可採 `<article>`。正確使用語義化標籤後，Google 搜尋結果之精選摘要（featured snippet）命中率可顯著提升。

#### （二）Flexbox 與 Grid 的使用時機判斷

CSS 兩大排版工具 Flexbox 與 Grid 雖皆可完成多數排版任務，然兩者之設計意圖不同，混用易導致樣式表膨脹與維護困難。判斷原則為「一維採 Flexbox，二維採 Grid」。Flexbox 適用於單一軸向之排列，例如頁首導覽列之橫向選單、按鈕組之縱向堆疊、表單欄位之標籤與輸入框對齊。Grid 適用於雙軸向之版面佈局，例如作品集九宮格、商品陳列頁、儀表板多區塊配置。教材第二講健身房網站之「課程卡片三欄」即為 Grid 之典型應用；若誤用 Flexbox 並以 `flex-basis: 33%` 模擬三欄，則於螢幕縮放時易產生溢位或換行異常。現代瀏覽器對兩者支援度均已成熟，不需額外考慮相容性問題。

#### （三）作品集網站的資訊架構

作品集型網站（第三講攝影師為代表，其後第十、第十五講亦同屬此類）之核心挑戰在於「如何在三秒內讓訪客決定要不要繼續看」。教材第三講提到「客人找不找得到你」與「找到之後看不看得懂你」兩項挑戰，本期補充三個資訊架構原則：第一，首頁須以最強作品起手，不應依時間序排列。攝影師之最佳作品與最近作品常非同一件，時間序呈現等於以「新」取代「好」。第二，作品分類應以觀者需求為準，而非作者之創作時期。婚紗攝影師可採「婚禮／孕寫／親子／商攝」分類，而非以「2023 / 2024 / 2025」分類。第三，每件作品頁應包含觀看時機、拍攝地點、費用區間三項資訊，以降低詢問門檻，此為教材未展開之補充。

### 三、實例分析：咖啡廳網站從 Template 到自訂品牌之三階段

觀察社區型咖啡廳之網站演進，可分為三個階段。第一階段為「套用 Template」。店家採用 Wix、Squarespace 等服務之現成模板，修改文字與照片即上線。此階段成本最低，但每個使用同一模板之店家視覺高度相似，品牌辨識度偏低。第二階段為「微調 Template」。店家請設計師或自行調整主色、字體與照片風格，使視覺與實體店面一致。此階段能凸顯品牌差異，但仍受模板結構限制，例如無法自訂訂位流程或會員系統。第三階段為「自行撰寫」。以教材所教之 HTML/CSS 直接建構頁面，配合 AI 協作完成互動功能。此階段工時最長，但獲得完整掌控權與後續擴充彈性。三階段無優劣之分，應依店家規模與預算選擇；但教材第一講之定位是「讓學員理解網站的每一層」，故選擇從第三階段切入，使學員不再受制於模板供應商。

## 貳、習題與解答

### 第一題【選擇題】

下列 HTML 結構中，何者**最適合**用於部落格之單篇文章？

(A) `<div class="post">...</div>`
(B) `<section class="post">...</section>`
(C) `<article class="post">...</article>`
(D) `<aside class="post">...</aside>`

**答：(C)**

**解析**：`<article>` 語義為「獨立、可轉貼之內容」，部落格單篇、新聞報導、作品集單件皆屬之。(A) `<div>` 無語義資訊，搜尋引擎與螢幕閱讀器無法辨識其為獨立文章。(B) `<section>` 適用於頁面內之主題性區塊，而非可獨立轉貼之單元。(D) `<aside>` 為側邊輔助資訊，非主內容。

### 第二題【選擇題】

下列排版需求中，何者**最適合**採用 CSS Grid 而非 Flexbox？

(A) 頁首導覽列：Logo 靠左、選單靠右
(B) 按鈕組：三顆按鈕橫向並排
(C) 作品集九宮格：三欄三列固定比例
(D) 表單欄位：標籤與輸入框左右對齊

**答：(C)**

**解析**：Grid 適用於雙軸向（列與欄同時控制）之佈局，九宮格為典型二維結構。(A)(B)(D) 皆為單一軸向之排列，Flexbox 更為簡潔適切。

### 第三題【選擇題】

攝影師作品集首頁之作品排序，下列何種策略**最能**提升訪客停留時間？

(A) 依拍攝日期由新至舊排列
(B) 以最強作品起手，其後依主題分類
(C) 依檔案名稱字母順序排列
(D) 隨機排列以增加驚喜感

**答：(B)**

**解析**：訪客於首頁停留之時間平均為 10 至 15 秒，決定是否繼續瀏覽之關鍵在於「前三件作品的品質」。(A) 時間序將「新」等同於「好」，常非實情。(C)(D) 皆未考量觀者之認知路徑。(B) 以最強作品建立第一印象，再以主題分類協助觀者快速定位所需類型，為業界通行做法。

### 第四題【是非題】

在 HTML 頁面中，所有區塊皆以 `<div>` 包覆，只要視覺排版正確即可，對 SEO 並無實質影響。（　）

**答：（×）**

**解析**：搜尋引擎爬蟲依賴語義化標籤判斷頁面結構與重要性。全部以 `<div>` 包覆之頁面雖視覺上可呈現，但爬蟲無法辨識何處為主內容、何處為側邊輔助、何處為頁尾版權資訊，將直接影響搜尋結果排名與精選摘要命中率。此外，螢幕閱讀器亦依語義化標籤提供導覽跳躍功能，影響無障礙使用者體驗。

### 第五題【是非題】

Flexbox 與 CSS Grid 功能重疊，於專案中擇一使用即可，混用會導致維護困難。（　）

**答：（×）**

**解析**：兩者設計意圖不同，應於同一專案中依需求混用而非擇一。Flexbox 適用於單軸排列（導覽列、按鈕組），Grid 適用於雙軸佈局（卡片九宮格、儀表板）。正確區分使用時機，反而能使樣式表更簡潔。混用之所以看似複雜，多因開發者未釐清兩者適用情境，將 Flexbox 套用於二維佈局或反之所致。

### 第六題【應用題】

試以 HTML 語義化標籤撰寫一段咖啡廳網站之首頁骨架，須包含頁首、主要內容（含「關於我們」與「菜單」兩個主題區塊）、側邊欄（營業資訊）與頁尾。

**答**：

```html
<!DOCTYPE html>
<html lang="zh-Hant">
<head>
  <meta charset="UTF-8">
  <title>巷口咖啡廳</title>
</head>
<body>
  <header>
    <h1>巷口咖啡廳</h1>
    <nav>
      <ul>
        <li><a href="#about">關於我們</a></li>
        <li><a href="#menu">菜單</a></li>
      </ul>
    </nav>
  </header>

  <main>
    <section id="about">
      <h2>關於我們</h2>
      <p>店家理念與創立故事...</p>
    </section>
    <section id="menu">
      <h2>菜單</h2>
      <article>
        <h3>手沖單品</h3>
        <p>產地、沖煮方式、價格...</p>
      </article>
    </section>
  </main>

  <aside>
    <h2>營業資訊</h2>
    <p>地址、電話、營業時間...</p>
  </aside>

  <footer>
    <p>&copy; 2026 巷口咖啡廳</p>
  </footer>
</body>
</html>
```

**解析**：此骨架符合語義化原則：`<header>` 包含品牌名稱與主導覽，`<main>` 標示主要內容區且每頁僅能有一個，其內以 `<section>` 分隔主題區塊並各自以 `<h2>` 為標題，菜單項目以 `<article>` 包覆以支援單獨轉貼，`<aside>` 為側邊輔助資訊，`<footer>` 為版權與聯絡資訊。每個區塊均設 `id` 以利導覽錨點連結。

### 第七題【應用題】

試撰寫一段 CSS，使用 Grid 將九件作品以三欄三列呈現，欄間距 20px，每件作品為正方形。

**答**：

```css
.portfolio {
  display: grid;
  grid-template-columns: repeat(3, 1fr);
  gap: 20px;
}

.portfolio-item {
  aspect-ratio: 1 / 1;
  overflow: hidden;
}

.portfolio-item img {
  width: 100%;
  height: 100%;
  object-fit: cover;
}
```

**解析**：`grid-template-columns: repeat(3, 1fr)` 建立三欄等寬佈局，`1fr` 表示依可用空間等比分配。`gap: 20px` 同時控制列間距與欄間距，無需另設 `row-gap` 與 `column-gap`。`aspect-ratio: 1 / 1` 強制每格為正方形，`object-fit: cover` 使圖片填滿格內而不變形。此寫法比 Flexbox 搭配 `flex-basis: 33.33%` 更為穩定，於螢幕縮放時不會因小數點誤差產生換行異常。

### 第八題【情境題】

一位剛入行的攝影師請你協助架設作品集網站，他希望「把所有拍過的照片都放上去」。請說明為何此做法不宜，並提出替代方案。

**答**：

此做法之問題有三。

第一，資訊量過大將降低訪客決策效率。研究顯示，選項超過 7 項時決策速度會顯著下降，超過 20 項則多數訪客會直接跳出頁面。若將所有作品（可能數百至數千張）全部呈現，訪客無法判斷哪些能代表攝影師的風格。

第二，將弱作與強作並列將拖累整體印象。訪客對攝影師之評價，常取決於其看到的「前五件作品」；若前五件中混入品質不一的作品，訪客會以品質較低者為心理基準。

第三，更新維護成本極高。若作品集無明確分類與排序原則，攝影師日後新增作品時將無法判斷應放於何處，最後往往放棄更新。

**替代方案**：採「代表作 + 分類作品集」之二層架構。首頁呈現 6 至 9 件精選代表作，其下以觀者需求為分類原則（如「婚禮／孕寫／親子／商攝」），每一類放 12 至 20 件。完整的歷史作品可另設「年度精選」頁面，分年度呈現，但入口不置於主導覽。

### 第九題【情境題】

某健身房老闆看到別家健身房的網站採用深色背景配霓虹色字體，認為「看起來很高級」，要求你也做成相同風格。請以設計思維之角度說明應如何回應。

**答**：

應回應之方向如下：

1. **視覺風格必須服務於品牌定位，而非單純模仿**。該健身房採深色霓虹可能因其客群為夜店客或 24 小時營業之都會客層；若本健身房之客群為社區媽媽或長輩，此風格將造成認知衝突。

2. **色彩直接影響使用者之生理反應**。深色背景配高飽和度字體雖具視覺衝擊力，但閱讀負擔較高，使用者平均停留時間較短。若網站目標是讓使用者詳細了解課表與費用，深色配色反而不利。

3. **提出診斷問題**：建議先釐清三件事——本健身房的主要客群是誰？希望他們在網站上完成什麼動作？競品網站的客群與目標與本店是否相同？釐清後再決定色彩策略。

4. **提出折衷方案**：若老闆仍偏好深色風格，可於「課程介紹」採白底易讀頁面，僅於「品牌故事」與「首頁 hero 區」採深色霓虹以營造氛圍，取得視覺衝擊與閱讀舒適之平衡。

設計不是老闆的喜好，也不是設計師的堅持，而是對使用者認知的回應。

### 第十題【問題與討論】

試比較「套用 Template」「微調 Template」「自行撰寫」三種網站建置方式之優缺點，並說明你會建議自己家人或朋友採用何種方式。請具體說明判斷依據。

**答（參考架構）**：

本題無標準答案，評分重點在於：

1. **是否理解三種方式的成本結構**（時間成本、金錢成本、學習成本、維護成本）；
2. **是否能依對象之條件給出差異化建議**（例如：家中長輩開早餐店 vs 朋友的新創公司 vs 自己的個人作品集，應採不同方案）；
3. **是否能說明長期考量**（Template 日後難以大改，自行撰寫則具擴充彈性）。

參考比較表：

| 方式 | 上線速度 | 品牌差異度 | 後續擴充 | 學習成本 |
|---|---|---|---|---|
| 套用 Template | 最快（1–3 天） | 低 | 受限 | 低 |
| 微調 Template | 中（1–2 週） | 中 | 中等受限 | 中 |
| 自行撰寫 | 慢（1–2 個月） | 高 | 完全自由 | 高 |

判斷依據可包含：家人時間精力、是否計畫長期經營、是否有商業擴張需求、是否願意投入學習等。評分不看結論，看推理過程。
