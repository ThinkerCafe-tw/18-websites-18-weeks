# AI 提示詞模板庫（工具無關版）

> 從第 1-8 講提取，標準化為「任何 AI 工具都能用」的格式。

## 使用說明（教材中應加入）

```
💬 以下提示詞可用於任何 AI 協作工具：
- ChatGPT（免費/付費）— 直接在對話框貼上
- Claude（免費/付費）— 同上
- Claude Code（進階）— terminal 環境，適合有經驗者
- 其他 AI 工具 — 只要支援程式碼生成即可

🔧 技術棧：React + Tailwind CSS + Supabase（第 4 講起）+ Vercel 部署
```

## 第 1 講：咖啡廳網站

### P1-1｜初始需求描述

**分類**：展示型網站 · 初始 brief
**目的**：用一段自然語言 + 截圖，讓 AI 生成完整四頁式網站

```
我想做一個咖啡廳網站，需要四個頁面。
首頁要有大圖橫幅和品牌介紹，
菜單頁要分類展示產品和價格，
關於頁面講我們的故事，
聯絡頁面有地圖。
整體風格要溫暖、簡約，像這些參考網站的感覺。
[上傳截圖]

請使用 React + Tailwind CSS 製作響應式網站。
```

### P1-2｜迭代優化（微調型）

**分類**：展示型 · 細節調整
**目的**：第一版出來後的快速修正

```
請做以下調整：
1. 首頁的字體可以大一點嗎？
2. 菜單的產品圖片想要圓角設計
3. 導航列改成深咖啡色背景
```

**教學重點**：不需要說「改 border-radius」，只要描述你想要的效果。

## 第 4 講：診所預約網站

### P4-1｜功能型網站初始 brief

**分類**：功能型 · 預約系統
**目的**：展示「從粗糙到完美」的四輪迭代法

**第一輪（建立原型）**：
```
我想做一個診所預約網站

【準備的資訊】
- 科別：內科、牙科、皮膚科
- 醫師：每科 2 位
- 診所名稱：仁愛診所
- 配色：專業的藍色系

請使用 React + Tailwind CSS 製作。
```

**第二輪（補充核心功能）**：
```
需要加上這些功能：

1. 首頁要有：
   - 診所橫幅照片
   - 「立即預約」的大按鈕
   - 三個科別的卡片展示
   - 醫師團隊介紹

2. 預約頁面要有完整流程：
   Step 1: 選擇科別（內科、牙科、皮膚科）
   Step 2: 選擇醫師（顯示醫師照片、姓名、專長）
   Step 3: 選擇日期（日曆介面，顯示當月和下個月）
   Step 4: 選擇時段（上午 9-12、下午 14-17、晚上 18-21）
   Step 5: 填寫資料（姓名、電話、Email、症狀）
   Step 6: 確認預約（顯示摘要）

3. 科別和醫師資料：
   - 內科：王大明醫師（消化系統專長）、李小華醫師（心血管專長）
   - 牙科：陳醫師、林醫師
   - 皮膚科：張醫師、黃醫師
```

**第三輪（優化 UX）**：
```
介面需要優化，請調整：

【日曆部分】
1. 今天用橘色邊框標示
2. 過去的日期變成灰色，不能點
3. 週日顯示「休診」並且不能選
4. 選中的日期用藍色底

【時段部分】
1. 按鈕要加大到 60px 高度（手機好按）
2. 每個時段顯示「尚有 X 個名額」
3. 已滿的時段顯示「已額滿」且變成灰色，不能點擊
4. 每個時段設定上限 10 人

【醫師選擇】
1. 醫師卡片要有照片（圓形頭像）
2. 顯示專長標籤（用淺藍色標籤）
3. 滑鼠移到卡片上時，要有陰影效果

【整體配色】
- 主色：#2563eb（藍色）
- 輔助色：#f97316（橘色，用於提示和按鈕）
- 背景：#f8fafc（淺灰）
```

**第四輪（打磨細節）**：
```
最後調整，讓系統更完整：

【表單驗證】
1. 電話欄位：只能輸入數字，必須是 10 碼
   - 如果不符合，顯示紅色提示「請輸入正確的手機號碼」
2. Email 欄位：必須包含 @ 符號
   - 格式錯誤時顯示「Email 格式不正確」
3. 所有必填欄位加上紅色星號 *

【預約成功頁面】
1. 顯示大大的綠色勾勾 ✓
2. 顯示「預約成功！」標題
3. 顯示預約編號（例如：A202511140001）
4. 完整的預約摘要

【手機版優化】
1. 日曆在手機上要可以左右滑動
2. 時段選項改成直式排列
3. 電話欄位自動彈出數字鍵盤
4. Email 欄位自動彈出帶 @ 的鍵盤
```

### P5-2｜建立會員資料表

**分類**：資料庫 · Schema 設計
```
我要建立一個美容院會員系統，使用 Supabase 作為資料庫。

【第一步：建立會員資料表】

請幫我建立一個名為 "members" 的資料表，包含以下欄位：

1. id — UUID，自動生成，主鍵
2. name — 文字，會員姓名，必填
3. phone — 文字，手機號碼，必填，唯一值
4. email — 文字，Email 信箱，選填
5. points — 整數，會員點數，預設值 50
6. total_spent — 整數，累積消費金額，預設值 0
7. created_at — 時間戳記，自動記錄建立時間
8. last_visit — 時間戳記，最後來店時間，可為空

請建立這個資料表，並顯示資料表結構讓我確認。
```

### P5-3｜建立會員註冊頁面

**分類**：功能型 · 表單 + 資料庫寫入
```
【第二步：建立會員註冊頁面】

需求如下：

【頁面設計】
- 視覺風格：粉紅色系（#ec4899 桃紅色），溫柔、專業、有質感
- 頂部：美容院 Logo + 「會員註冊」標題
- 底部：「已經是會員？點此登入」連結

【表單欄位】
1. 姓名：必填，2-10 個字
2. 手機號碼：必填，10 碼數字（09 開頭），即時驗證
3. Email：選填，有填要檢查格式

【註冊邏輯】
1. 驗證所有必填欄位 + 格式
2. 檢查手機號碼是否已註冊（查 Supabase members 表）
3. 寫入 Supabase：name, phone, email, points=50, total_spent=0
4. 成功：顯示「恭喜您成為會員！已贈送 50 點歡迎點數」

【手機版優化】
- 按鈕至少 44px 高
- 電話欄位→數字鍵盤，Email→帶 @ 鍵盤
- 表單寬度：手機 90%，電腦最寬 400px

請使用 React + Tailwind CSS，連接 Supabase。
```
