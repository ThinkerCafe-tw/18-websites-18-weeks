# 第 13 講：汽車保養網站

一個對機械陌生的車主，最怕的不是車壞，是站在報價單前不知道哪一項是真的、哪一項是被坑。本講就用 AI（Artificial Intelligence，人工智慧）協作做出一個汽車保養網站原型，把查詢服務項目、預約保養、記錄車籍與下次回廠提醒，組成一個讓他不必懂機械也敢把車交出去的數位入口。

本講完成後，你會得到四個成果：

- 一份汽車保養網站需求摘要。
- 一個可展示的保養預約與提醒網站原型。
- 一份涵蓋車主、車籍、里程、服務、預約、紀錄與提醒時間的資料模型草稿。
- 一份手機版、信任證據、流程斷點與上線前檢查清單。

本講學習重點：

◆ 理解車主的核心恐懼：不是機械問題，是失去對生活的掌控感
◆ 掌握保養廠三條商業邏輯：零件毛利、預防性里程管理、免費健檢轉化
◆ 用信號理論與損失規避，解釋技師如何在定價上超越原廠
◆ 認識三種黑暗手法，清楚法律責任與行車安全的雙重代價
◆ 學會把本講產業需求轉成頁面模組、資料模型、AI 提示詞與驗收清單

## 1. 破冰案例

**本講情境：週六早上七點五十分，停車格裡**

週六早上七點五十分，新北市某社區地下停車場。一輛黑色轎車，五年前買的，保固期在上個月正式結束。

車主站在駕駛座旁，鑰匙沒有插進去。車子昨晚發動時有震動，不明顯，但有——一種細小、不正常的頻率，從方向盤傳上來，紅燈停下時最清楚，一踩油門又好像沒了。他不確定自己是不是太敏感。

他打開 Google Map 輸入「汽車保養」開始滑。第一家照片是油汙油亮的大廠，評論第一條：「便宜，但叫你換很多東西，不知道有沒有必要。」第二家評論沒幾條，停車場太小，車恐怕塞不進去。第三家照片裡一排技師穿著同色系工作服，背景是乾淨的地板，最新評論寫：「師傅修到一半 LINE 我傳照片，說哪個零件壞了，新舊件擺一起讓我自己看。從來沒有被這樣對待過。」

他點進去，把電話存下來。他不知道這家廠技師幾年資、拿什麼證照，但那個傳照片的動作，讓他覺得這裡的人願意讓他看。

---

關鍵細節是：他沒有在想「這家技術好不好」。他在找的線索不是技術認證，是「哪一個具體動作，讓我覺得這裡的人把我當成人而不是案子」。本講從這個線索出發。

**保養廠賣的從來不是技術——那是最低入門門檻，不是賣點。賣的是「讓一個對機械完全陌生的人，在這裡不會有被騙的感覺」。**

## 2. 業態需求

**產業聆聽：他在評論區裡在找什麼**

#### 三個核心恐懼

車主進廠之前，心裡已跑完一份恐懼清單：

**恐懼一：資訊不對稱下的「被坑殺感」。** 他不懂機械，技師說「這個要換」他無力反駁也無從確認，於是進入防禦模式——任何追加都被詮釋成「剛才是假的」。他防禦的不是錢，是「被看扁」。

**恐懼二：零件替換的黑箱疑慮。** 他知道「副廠件充原廠」存在卻無從驗證；念頭一旦種下，每張報價單的零件品名和價格都會多一層懷疑。

**恐懼三：維修瑕疵帶來的生命威脅。** 這條最少被說出口卻最深。「修好這個、壞那個」的故事他聽過，高速公路熄火的畫面也出現過——它涉及家人也坐在車上，只要一個影像就存在。

#### 三個猶豫點

**報價的隨意性。** 對方說「沒看車沒辦法說」他理解，卻沒有安全感；他要「就算追加也會先跟我說」的廠。

**廠房的視覺混亂度。** 他用照片裡地板的乾淨程度做替代性指標：「連地板都不管，修車會不會也隨便？」推論未必成立，但大腦就是這樣。

**保固承諾的效力感。** 口頭說有保固卻沒白紙黑字，他無法判斷算不算數；他要一份看得懂、有日期品項簽名的記錄。

#### 三個信任觸發

讓他最後決定進廠的，通常是其中之一：

**全透明的「LINE 維修直播」。** 技師拆解後主動拍照、新舊件並排傳給車主，成本三十秒，傳遞「我不怕你看」。

**軟體生成的報價單與維修紀錄。** 數位化工單分列品項、數量、單價、工資，讓他覺得「這家廠有制度」。

**「先報價、後動工」的嚴格執行。** 超出討論範圍的項目動手前先確認，把決策權還給車主。

#### 人性真相

**客戶買的是「對生活的掌控權」。他不是在找最便宜的技師，是在找能最快讓他回到正常生活的人。** 技師若能說「今天下午兩點可以看，約兩小時完工」，他對維修費用的敏感度會大幅下降。

### 2.1 產業營運邏輯

**產業心法：老手是怎麼算這門生意的**

#### 零件毛利才是真正的利潤引擎

工資受公會牌價與市場行情壓縮，零件不一樣。以空氣濾網為例：OEM（Original Equipment Manufacturer，原廠委託製造）件材料成本約一百二十到一百八十元，市場收費卻從四百到八百元都有，而消費者幾乎沒有參照點。老手以「比原廠便宜」維持百分之一百五十到兩百以上的毛利。結構上，有競爭力的保養廠零件收入應佔整體營收六成以上。

**工資讓廠家活著，零件讓廠家賺錢，老手從第一天就把這兩個數字分開計算。**

#### 預防性更換與「里程管理」邏輯

技師的邏輯是在車壞之前告訴車主它快壞了。這是一套系統：老手為每輛車建檔（車型、年份、里程、上次換過什麼、下一個該注意的），在胎深或電瓶逼近臨界時主動提出，說「安全性」而非「你要花錢」。效果上，車主每年基礎保養費用約四千到六千元，若持續追蹤並適時建議預防性更換，年度金額可推升至兩萬元以上、差距四到五倍。這套邏輯的底層基礎設施，是真實、可追蹤的數位化車輛資料。

#### 免費健檢與「問題轉化率」

「春季安全健檢」是保養廠最強的獲客工具之一。成本是技師二十到三十分鐘工時，回報是讓車主面對儀器出具的白紙黑字數據——「電瓶充電效率剩百分之六十八」是儀器的說法，車主對儀器的抵抗力遠低於口頭說詞。執行良好的廠家，健檢後的「建議維修轉化率」可達五成到六成以上。

因此本講的技術主題是「工單資料與週期提醒」。前述三條商業邏輯都建立在可追蹤的車輛資料之上；第 4 節會把這層基礎拆成車籍、工單、估價與提醒的資料模型，第 5 節再讓 AI 把這些結構落成可檢查的網站原型。

## 3. 網頁設計重點

### 3.1 網站風格

風格要讓車主感到透明、可靠、可追蹤。照片應呈現廠內環境、工位、工具、零件、技師與完工交車流程，而不是只放跑車或改裝照。車主多半不是喜歡修車才來，是不確定車況；網站要降低資訊不對稱，而不是放大恐懼。

### 3.2 建立信任

信任模組要包含工單、估價單、零件來源、施工照片、保養紀錄與下次提醒；若有原廠件、副廠件或翻修件，應清楚標示差異與價格。最重要的不是喊便宜，而是讓車主知道每項工事為什麼需要、是否急迫、是否可延後。

### 3.3 引導行動

主要行動是預約保養、填寫車籍資料、查看項目、確認估價、設定里程提醒。CTA（Call to Action，行動呼籲）可依情境設計，例如「我要定期保養」「我有異音想檢查」「查詢上次紀錄」。

### 3.4 心理學錨點

**車主不理性，但他的不理性有規律**

#### 信號理論：全透明是對作弊最有力的抑制

一九七三年，麥可．史賓賽（Michael Spence）在《經濟學季刊》（*Quarterly Journal of Economics*，第八十七卷第三期）發表後來獲諾貝爾獎的「信號理論」：資訊不對稱時，賣家要讓買家相信品質，就得提供「作弊成本極高」的信號。全透明的「LINE 維修直播」對誠實技師成本幾乎為零，對打算換劣質件的技師卻是一道過不了的門——這就是「傳照片」能建立信任的機制。

#### 損失規避：修車的時機感非常不對稱

一九九一年，阿莫斯．特沃斯基（Amos Tversky）和丹尼爾．康納曼在《經濟學季刊》（*Quarterly Journal of Economics*，106(4), 1039–1061）確認：人對「損失」的痛苦，大約是對「等值獲得」喜悅的兩倍。這在維修場景特別明顯——「換輪胎要一萬元」讓車主覺得貴，但「繼續開，高速爆胎、車毀人傷，修車費加醫療費可能十倍以上」的畫面，讓他幾乎無法再猶豫。老手不是在嚇車主，是把同一筆支出放進「避免損失」框架，接受度明顯高於「獲得好處」框架。

---

### 3.5 信任底線

**三種讓你無法面對自己鏡子的手法**

#### 底線一：刻意製造故障

技師刻意讓避震器油封或冷卻水管在健檢後短期內劣化，幾週後客戶帶著「剛保養完卻漏油」回廠，再順理成章地收一筆。這不只是欺詐，更直接威脅行車安全——避震器失效影響穩定性、冷卻水問題導致引擎過熱。台灣《消費者保護法》對故意損害最高可請求三倍懲罰性賠償；若造成車禍傷亡，刑事責任也完全存在。但真正的問題是：你一輩子要帶著這件事睡覺。

#### 底線二：以副廠件充原廠，按原廠收費

工單寫「原廠火星塞，三千六百元」，實際裝上的是成本不到兩百元的無品牌翻新件；正時皮帶、水泵、點火線圈這類「更換頻率低、單價偏高、外觀難辨別」的零件最常見。問題不只是欺騙金錢：正時皮帶斷裂的後果是引擎報廢，而且發生當下車主在路上。

#### 底線三：幽靈保養與虛報工項

「積碳清洗、燃油保護劑、底盤防鏽」這類工單品項，有些廠家只是加幾瓶添加劑、或用抹布擦了引擎蓋——客戶進不去施工區，也不知道要驗收什麼。一條「加了四千塊積碳清洗但引擎燈還是亮著」的評論，乘以 Google Map 的可見性，半年的客源衝擊是可以計算的。

---

**我們知道這些手法存在，但選擇不用。你這輩子要做的不是「有效的生意」，是「十年後還抬得起頭的生意」。**

---

## 4. 功能與資料模型

**實作主題：把「客戶資料與回流設計」落成可檢查的網站。**

前面談的是產業、人性與風險；這一層要把判斷轉成網站結構。AI 協作不是請它「做一個漂亮網站」，而是把業態需求、頁面模組、資料欄位、流程規則與驗收標準說清楚，再讓它產出第一版。

### 本講網站需求

本講的網站型態是「保養預約與提醒網站」，核心行動是查詢服務項目、預約保養、記錄車籍與下次回廠提醒，需求整理如表 13-1。

**表 13-1　汽車保養網站的需求模組**

| 模組 | 功能 | 必要內容 |
|---|---|---|
| Hero 首頁 | 建立第一印象 | 名稱、定位、主張、主要行動按鈕 |
| 信任證據 | 降低猶豫 | 真實照片、資歷、評論、規則或案例 |
| 核心流程 | 完成主要任務 | 查詢服務項目、預約保養、記錄車籍與下次回廠提醒 |
| 資料表單 | 收集必要資訊 | 姓名、聯絡方式、情境欄位、備註 |
| 規則說明 | 避免誤解 | 價格、限制、取消或退款規則 |
| 聯絡與後續 | 接住下一步 | 電話、LINE、Email、地圖或確認訊息 |

### 頁面與功能模組

把網站想成一組可檢查的模組，每個模組回答一個使用者問題：我在哪裡？這適合我嗎？我相不相信？下一步做什麼？最小可行版本可先做單頁，但不代表沒有結構——導覽列能跳到主要區塊、主要按鈕出現在第一屏與結尾、表單送出後要有確認訊息。

### 資料與流程

本講至少要先拆出下列資料物件，如表 13-2。第一版不一定要接資料庫，但你必須知道網站背後需要哪些資料，AI 才不會只產出表面文案。

**表 13-2　汽車保養網站的基礎資料物件**

| 資料物件 | 欄位 | 範例或用途 |
|---|---|---|
| `AutoServiceProfile` | 名稱、地區、定位、營業或服務時間 | 用於頁面內容、表單或狀態檢查 |
| `UserRequest` | 姓名、聯絡方式、需求、備註 | 用於頁面內容、表單或狀態檢查 |
| `ServiceItem` | 名稱、內容、價格或條件、限制 | 用於頁面內容、表單或狀態檢查 |
| `ScheduleSlot` | 日期、時段、名額、狀態 | 用於頁面內容、表單或狀態檢查 |
| `Policy` | 取消、退款、隱私或服務邊界 | 用於頁面內容、表單或狀態檢查 |
| `TrustEvidence` | 照片、評論、資歷、案例或證明 | 用於頁面內容、表單或狀態檢查 |

流程可以用這條線檢查：

```text
看見首頁 → 判斷是否適合 → 查看信任證據 → 進入核心流程 → 填寫必要資料 → 看到確認或下一步
```

如果中間任何一步需要人用 LINE 或電話補洞，就要在網站上說清楚，不要讓網站假裝有系統能力、實際只收一張表單。

### 車籍、工單與里程提醒

汽車保養網站不是只列服務項目。真正能建立回流的是車籍資料、保養紀錄與下次提醒：車主不會記得上次換什麼油、幾公里換煞車皮、下次該不該檢查輪胎，保養廠若記得，就比只會報價的店更可信。資料模型可拆成下列物件，如表 13-3。

**表 13-3　車籍、工單與提醒的資料模型**

| 資料物件 | 重要欄位 | 用途 |
|---|---|---|
| `VehicleProfile` | 車牌、品牌、車型、年份、目前里程 | 建立車籍 |
| `CustomerProfile` | 車主姓名、電話、Email、聯絡偏好 | 聯絡與提醒 |
| `ServiceMenu` | 保養項目、參考價格、所需時間、適用車型 | 對外服務說明 |
| `Appointment` | 日期、時段、車輛、服務項目、狀態 | 預約管理 |
| `WorkOrder` | 檢查項目、建議維修、車主同意狀態、完工狀態 | 現場工單 |
| `EstimateLine` | 零件、工資、數量、單價、是否同意 | 報價透明化 |
| `MaintenanceRecord` | 日期、里程、完成項目、照片、下次建議 | 保養紀錄 |
| `ReminderRule` | 里程門檻、時間間隔、提醒內容 | 下次回廠提醒 |

保養流程可以用這條線理解，如圖 13-1 所示（請排版將下列步驟製成橫式流程圖，「車主不同意」為報價確認後的分支）。

**圖 13-1　從預約到下次提醒的保養流程**

```text
預約 → 進廠檢查 → 報價確認 → 施工 → 完工交車 → 下次提醒
        │
        └→ 車主不同意 → 僅完成已同意項目
```

這裡最重要的是「報價確認」：保養廠不能在沒有車主同意下自行加項目，否則信任一次破裂。網站原型可以示範報價單，如表 13-4。

**表 13-4　報價單示範**

| 項目 | 零件 | 工資 | 小計 | 狀態 |
|---|---:|---:|---:|---|
| 機油更換 | 1800 | 500 | 2300 | 已同意 |
| 空氣濾芯 | 600 | 0 | 600 | 待確認 |
| 煞車皮 | 3200 | 1200 | 4400 | 車主暫緩 |

這種表格比「價格透明」四字更有說服力。

### 保養紀錄與下次提醒

汽車保養的長期價值在於回流，網站應示範如何記錄下次提醒，如表 13-5。

**表 13-5　下次提醒的類型與觸發條件**

| 提醒類型 | 觸發條件 | 例子 |
|---|---|---|
| 里程提醒 | 距離上次保養 5000 公里 | 機油與濾芯檢查 |
| 時間提醒 | 距離上次保養 6 個月 | 定期保養 |
| 零件壽命 | 煞車皮、輪胎、電瓶 | 上次檢查已接近更換 |
| 年檢提醒 | 到期前 30 天 | 驗車與保險提醒 |

提醒文字要避免恐嚇，以「上次紀錄」為據：

```text
上次保養紀錄顯示您的車輛里程為 52,000 公里，建議在 57,000 公里左右檢查機油與濾芯。若近期已在其他地方保養，可忽略此提醒。
```

這句話的信任感來自有紀錄、有彈性、沒有威脅。

## 5. AI 協作實作

本講的 AI 協作分兩層：先讓 AI 產出網站，再檢查產出、必要時請 AI 補強。本節依「提示詞 → 產出結果 → 檢查修改 → 保存部署」四個小節進行。

### 5.1 本講提示詞

可以用下面這份提示詞作為第一版。

```text
你是網頁設計老師與前端工程師。
請幫我做成一個可以直接打開的「汽車保養網站」單頁原型。

【網站目標】
服務項目、預約、車籍資料、里程提醒與透明報價

【使用者情境】
使用者第一次進入網站時，需要在手機上快速判斷這個服務是否適合自己，並能完成主要行動：查詢服務項目、預約保養、記錄車籍與下次回廠提醒。

【必要區塊】
1. Hero 首頁：名稱、定位、主張、主要行動按鈕
2. 信任證據：真實照片、資歷、評論、規則或案例
3. 核心流程：查詢服務項目、預約保養、記錄車籍與下次回廠提醒
4. 資料表單：姓名、聯絡方式、情境欄位、備註
5. 規則說明：價格、限制、取消或退款規則
6. 聯絡與後續：電話、LINE、Email、地圖或確認訊息

【資料欄位】
請在頁面中示範下列資料結構：
1. AutoServiceProfile：名稱、地區、定位、營業或服務時間
2. UserRequest：姓名、聯絡方式、需求、備註
3. ServiceItem：名稱、內容、價格或條件、限制
4. ScheduleSlot：日期、時段、名額、狀態
5. Policy：取消、退款、隱私或服務邊界
6. TrustEvidence：照片、評論、資歷、案例或證明

【設計要求】
1. 請用清楚的 HTML（HyperText Markup Language，超文字標記語言）結構與 CSS（Cascading Style Sheets，階層式樣式表），建成一個可以直接打開的單一 index.html，CSS 寫在同一個檔案裡。
2. 請使用 RWD（Responsive Web Design，響應式網頁設計），手機、平板、桌機都要可讀。
3. 請避免假評價、假資歷、假價格或過度承諾。
4. 表單與按鈕要有清楚的互動狀態，可以用靜態示意，不需要真的送出到伺服器。
5. 
```

### 5.2 AI 產出結果

把這份提示詞交給你選的 Vibe Coding 工具，它會直接幫你在資料夾裡建好 `index.html`——把車主的查詢、預約、車籍與里程提醒一次落成頁面。用瀏覽器打開那個檔案，就會看到類似圖 13-2 的保養廠首頁。

**圖 13-2　AI 依本講提示詞產出的汽車保養網站首頁（桌機版）**

（圖檔：`fig-13-02-home-desktop.png`，隨稿附件）

首頁第一屏包含店名、地區與定位、一句話主張「先報價、後動工」，以及「查詢服務項目」「預約保養」兩個按鈕，按鈕下方標出營業時間；往下依序是信任證據、服務項目、預約保養、車籍紀錄與下次提醒、規則說明、聯絡與後續。廠內照片與地圖先以免費圖庫的示意圖代替，AI 也用註解標出了日後要替換的正式檔名，例如 `images/garage-01.jpg`。

值得注意的是，信任證據區塊放的是「先報價、後動工」「施工照片即時回傳」「看得懂的紀錄」三條做事規則與廠內照片示意圖，沒有出現編造的客戶評價與資歷，這是提示詞設計要求第 3 點在發揮作用。

這一版產出的內容可以整理成一份清單：

- 一個 `index.html`，HTML 與 CSS 寫在同一個檔案內，`<title>` 與 meta description 放入了店名、地區與服務內容，適合搜尋引擎理解。
- `header`：店名與導覽。
- `main` 內依序是主視覺、信任證據、服務項目、預約保養（含本週時段示意與預約表單）、車籍紀錄與下次提醒、規則說明、聯絡與後續七個 `section`。
- `footer`：店名、營業時間、服務範圍與版權聲明。

整體結構可以用下面的骨架示意（省略號代表實際內容，不是可直接貼用的程式碼），這就是表 13-1 的需求模組落到程式碼後的樣子。其中表 13-1 的「核心流程」被 AI 拆成服務項目、預約保養、車籍與提醒三個 `section`，是常見而可接受的拆法：

```html
<header>…店名與導覽…</header>
<main>
  <section class="hero">…名稱、定位、主張、查詢服務項目、預約保養…</section>
  <section id="trust">…三條透明守則與廠內照片示意圖…</section>
  <section id="services">…服務項目、參考價格、所需時間、限制…</section>
  <section id="booking">…本週時段示意與預約表單…</section>
  <section id="records">…保養紀錄示意、提醒類型與提醒範例…</section>
  <section id="policy">…價格、取消、退款與個資規則…</section>
  <section id="contact">…電話、LINE、Email、地圖與後續說明…</section>
</main>
<footer>…店名、營業時間、版權…</footer>
```

手機版的關鍵是 CSS 裡的 media query。AI 產出的規則節錄如下：

```css
/* 平板：服務與信任卡片改為兩欄 */
@media (max-width: 1024px) {
  .grid { grid-template-columns: repeat(2, 1fr); }
}
/* 手機：全部單欄，header 改為直向排列，按鈕與表單欄位維持易點 */
@media (max-width: 640px) {
  header { flex-direction: column; align-items: flex-start; gap: 0.4rem; }
  .grid { grid-template-columns: 1fr; }
}
```

用手機寬度打開同一個檔案，服務項目會改成單欄卡片，預約表單欄位也會整排可點，如圖 13-3。

**圖 13-3　同一個檔案在手機寬度下的單欄服務項目**

（圖檔：`fig-13-03-mobile-services.png`，隨稿附件）

把產出結果對回第 2 節的業態需求：分列零件與工資的估價規則，回應的是「資訊不對稱下的被坑殺感」；施工照片回傳與新舊件並排核對，就是第 3 節信號理論說的高作弊成本信號；車籍紀錄與下次提醒，回應的是車主真正要買的「對生活的掌控權」；可預約時段表，回應的是「我今天下午兩點可以看」的時間確定性。對不回去的區塊，就是可以刪掉的裝飾。

這裡要先打個預防針：AI 產出的版面本來就帶點變化，你做出來的和圖 13-2 不會完全一致，這很正常。真正要驗收的從來不是「長得一樣」，而是下一小節逐項列出的檢查點。

### 5.3 結果檢查與修改

AI 產出後不要只看漂不漂亮，請依序檢查：

- 首頁第一屏是否說清楚服務適合誰與下一步。
- 是否收集車籍、年份、里程與服務需求以支撐核心流程，而非只留姓名電話。
- 是否有工單與報價明細，能區分零件、工資與車主同意狀態。
- 是否能記錄保養完成項目與下次提醒條件。
- 是否避免恐嚇式語言推銷維修。
- 價格、限制、取消或隱私規則是否清楚。
- 手機版是否能完成主要行動，且文字與按鈕不重疊。

### 5.4 保存與部署

要上線可以先用 GitHub（程式碼與專案託管平台）保存檔案，再用 Vercel（前端部署平台）或 GitHub Pages 部署。GitHub 讓每次修改都有紀錄；部署不是最後才做的事，是讓你用真實手機檢查網站是否可用。

## 6. 上線與延伸

本講最重要的練習，是把「產業洞察」翻譯成「可檢查的網站需求」。當你能說清楚資料、流程、信任與風險，AI 才會從模板生成器變成真正的協作夥伴。

---

### 6.1 重要名詞

**OEM 件**：符合原廠規格、授權供應商生產的零件，品質近原廠件而售價較低，與無品牌翻新件有本質區別。

**預防性維修**（Preventive Maintenance）：不等零件失效，依里程或儀器診斷提前更換，降低拋錨風險、提升年度保養金額。

**里程追蹤 CRM（Customer Relationship Management，客戶關係管理）**：記錄每輛車歷史維修與里程、適時提醒回廠的系統，可從 Google 試算表起步。

---

### 6.2 自我評量

**第一題（是非）**：車主在選擇保養廠時，最主要的評估標準是技師的技術證照數量。

**第二題（是非）**：有競爭力的保養廠，零件收入佔比通常高於工資收入，因為工資受市場行情壓縮、零件毛利空間較大。

**第三題（選擇）**：根據信號理論，哪個做法是「作弊成本最高」的信任信號？
（甲）廠門口掛上所有技師的證照框
（乙）維修中主動 LINE 傳零件拆解照片，新舊件並排給客戶看
（丙）官網刊登「絕不使用副廠件」聲明
（丁）提供三個月保固承諾

**第四題（選擇）**：哪一個最符合「損失規避」的正確應用？
（甲）「現在保養可以省錢」
（乙）「現在換胎比預期便宜」
（丙）「胎紋已達臨界值，雨天煞車距離延長三成，爆胎後修車和醫療費遠超過現在換胎」
（丁）「其他客人都換了，你要不要也換」

**第五題（是非）**：「免費春季健檢」對保養廠是純公益服務，無法直接創造收入，只能提升品牌形象。
