# 教材 v2 寫作準則（17 講共用）

## 核心定位（最高層級約束）

這**不是**「教做網站」的書，是**18 個服務業的商業心法書**，附贈做好的網站。
《卡內基銷售學》讀完記得的是人性洞察，不是話術 SOP。這本書走這個模子。

**比例鐵律**：L1-L4（聆聽/心法/心理學/警示）佔 80%；L5（Vibe Coding）佔 20%。

## 6 層結構與字數預算（純中文 CJK 計算）

| 區塊 | 字數 | 說明 |
|---|---|---|
| 單元目標 | 100-150 | 4-5 條，◆ 開頭 |
| 破冰 | 600-800 | 微距小說感場景，不教技術 |
| L1 產業聆聽 | 700-900 | 三恐懼/三猶豫/三觸發 + 人性真相金句 |
| L2 產業心法 | 800-1000 | 三條明道，每條都有具體數字 |
| L3 心理學錨點 | 500-700 | 2-3 個原理，引用學術文獻 |
| L4 人性警示 | 500-700 | 三警示，含手法+心理學+法律+傷害 |
| L5 動手（Vibe Coding） | 1200-1500 | 提示詞+現成 HTML 交付流程 |
| 重要名詞 | 150-350 | 5-8 個本講關鍵詞定義 |
| 自我評量 | 300-500 | 5-7 題（是非/選擇/情境） |
| 參考文獻 | 50-200 | APA 格式 |
| 延伸學習 | 100-200 | 推薦書/網站 |

**總字數目標**：5,500-7,000 純中文字（華視規格嚴格，總字數超 7,200 即需裁剪）

**字數上限是硬上限不是建議**：每一區塊的上限是硬上限。若 L1 規定 700-900，絕對不可寫到 1,400。超標即需在同一區塊裁剪到範圍內再交付。內容豐富不是超標的理由，精煉才是。

**L3 語氣警告（從 Wave 1 學到）**：引用學術文獻時，要保持敘事感與口語感。不要寫成「作者（年份）在《期刊》發表⋯⋯研究方法為⋯⋯結論指出⋯⋯」這種論文格式。學術是錨點，不是講稿。好的 L3 語氣示範已在第 01 講 v2，請嚴格對齊。

## 硬約束（違反重寫）

1. **繁體中文 + 台灣用詞**
2. **禁用工程師腔**：OKR、KPI、ROI、CVR、UX、roadmap、痛點、賦能、閉環、顆粒度、對齊、抓手、落地、承接、交付（在 L5 可用「交付」但不濫用）
3. **禁用流行過時詞**：智能助理、智慧助理、無程式、零程式、元宇宙、Web3、革命性、顛覆、一鍵搞定
4. **禁用 AI 腔**：「讓我們一起」「在數位時代」「前所未有」「隨著科技發展」「放眼未來」「總的來說」「綜上所述」「親愛的讀者」「各位學員」
5. **禁用空泛成語排比**（「見微知著、舉一反三」這類）
6. **引用精確**：學術引用寫作者、年份、期刊、DOI；官方數據寫來源機構 + 文件編號或網址
7. **每層至少一句金句**（可摘錄、可傳誦、有記憶點）
8. **不誇大商業數字**：每個百分比/金額要有出處或標「業界觀察」「示意假設」
9. **不寫 Emoji**（單元目標用 ◆ 除外）
10. **不寫「上一講我們學了」**；結尾可一句提下一講，不超過 30 字
11. **破冰小說感**（微距鏡頭、具體時間地點人物）、不用「各位」「親愛的」

## 語氣示範（嚴格對齊以下聲音）

**破冰調**：
> 週三下午兩點，信義區。一個背著後背包的人站在巷口滑手機。他沒有在想「這家咖啡好不好喝」— 他在想「我如果進去坐四小時，店員會不會給我臉色」。

**產業心法調**：
> 150 元拿鐵除以四小時 = 每小時坪效 37.5 元，這連信義區租金的十分之一都不夠。老手知道這件事，他們接受這件事，但老手同時在做三件事讓帳最終是正的。

**心理學錨點調**：
> 1975 年的客人決定要不要走進你的店，可能要想 15 分鐘。2026 年的客人只給你 50 毫秒。

**人性警示調**：
> 我們知道這些手法存在。我們選擇不用。你這輩子要做的不是「有效的生意」，是「能在十年後還抬得起頭的生意」。

**L5 收尾調**（每講 L5 要有類似收尾）：
> 一個網站不會讓你的咖啡廳更好。但一個正確理解客人在害怕什麼的老闆，會。

## L5 統一模板（所有行業共用骨架、但提示詞要客製化）

### L5 5 步驟骨架

1. **打開教材配套網站**：`thinker.cafe/cafe-game/lessons/{NN}`（NN = 該講編號，如 02、03...）
2. **複製提示詞範本**（完整可複製、三層結構：上下文+需求+風格）
3. **貼到 AI 助理**（ChatGPT / Gemini / Claude 任選）
4. **只改三件事**（該行業對應的三要素，如診所改「診所名/醫師簡介/服務項目」）
5. **部署（選做）**：Vercel / Netlify / GitHub Pages 三選一，一句話帶過

### L5 結尾必帶金句（每講都要有這層轉折）

格式：「一個網站不會讓你的 {行業} 更好。但一個正確理解 {該行業客戶在怕什麼} 的 {老闆/專業人士}，會。」
