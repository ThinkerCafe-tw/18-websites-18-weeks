## 🔴 嚴重問題（必修）

### 1. 第五講有巨大「失控代碼塊」吞掉整個後半章
**位置**：`book-preview.html:1685–2005`（章節 5-4 測試與驗證 → 重要名詞 → 自我評量 → 參考文獻/書目 全部被包進一個 `<pre><code>`）

**現象**：從 `<pre><code>` 開始（1685 行）一路到 `</code></pre>`（2005 行），整段 320 行的教學內容（含「八、完整測試流程」「九、本講完成了什麼」「十、下週預告」「十一、延伸思考」「重要名詞」「自我評量」「參考文獻」）全部以 monospace 顯示在灰底 code block 裡，**所有 `**粗體**`、`###` 標題、`◆` 名詞符號都是 raw markdown**。後續的 `<p>□ 填寫完整資料...</p>` 清單被切散成好幾個短短的 `<pre><code>` 片段，章節完全破碎。

**根因**：`01-教材內容/第05講-美容院會員網站(上).md` 裡有**落單的 ``` 圍籬**（某個 code fence 沒成對關閉，或章節中用了單一 ``` 當裝飾線），`md_to_html()` 的 `in_code` 狀態從那一刻起卡住到下一個 ``` 才復位。

**修法**：
1. 打開第 5 講 md 原檔，用 `grep -n '^```' 第05講-美容院會員網站\(上\).md` 數出 ``` 是否成對（奇數就爆）。
2. 補上缺失的 ``` 或刪掉多餘的一個。
3. `build-book.py` 加個護欄：每章結束時若 `in_code=True` 就印 warning + 強制 `</code></pre>`，並在 stderr 報錯章節名，避免未來再發生同類事故無聲傳播。
4. 重跑 `python3 build-book.py`，確認第 5 講恢復正常段落。

### 2. 目錄/章節標題的「冒號」全半形不一致
**位置**：目錄 `book-preview.html:226`；每章 `<h1>`

**現象**：
- 使用全形冒號「：」：第一、三、四、七、九、十、十一、十三、十五、十六、十七、十八、十九講
- 使用半形冒號「:」：第五、六、八、十二、十四講

交叉對照源檔名（`第05講-美容院會員網站(上).md`、`第08講-餐廳訂位網站.md` 用 `(` 半形括號，其他講次用「：」），問題從源檔就有。這是正式出版的**視覺噪音 + 校對硬傷**。

**修法**：批次把所有章節 `##` 一級標題統一為全形「：」「（上）」「（下）」。可寫一個 normalizer 在 `md_to_html` 前對 title 做 `.replace(':', '：').replace('(', '（').replace(')', '）')`，同時修原檔。

### 3. 有序列表（1. 2. 3.）完全沒轉成 `<ol>`
**位置**：全書 456 處

**現象**：`build-book.py` 的 `md_to_html` **只處理了 `◆ - *` 開頭的無序清單**，沒寫 ordered list 分支。結果所有「1. xxx / 2. yyy / 3. zzz」題號全部變成**獨立的 `<p>1. xxx</p>`**，每條題目之間被 `margin: 0.6em` 撐開，**沒有縮排、沒有連續感、沒有自動編號結構**，看起來像隨機散落的句子。特別是：
- 自我評量的考題（第 5, 19 等講尾）
- 參考文獻編號
- 「三步驟」「五步法」教學

**修法**：在 `md_to_html()` 的 bullet 分支前加：
```python
om = re.match(r'^(\d+)\.\s+(.*)', stripped)
if om:
    if not in_ol:
        out.append("<ol>"); in_ol = True
    ...
```
同時在每次換段 / heading / code / hr 時關閉 `in_ol`，跟 `in_ul` 同機制。CSS 補 `ol { margin: 0.5em 0 0.5em 1.5em; } ol li { margin-bottom: 0.25em; }`。

### 4. 截圖佔位符 regex 太嚴格，大量 `[截圖 ...]` 被當普通文字
**位置**：全書只有 19 個 `.screenshot-placeholder`（集中在第 9、11、14、15 講），其他 16 講**零截圖框**

**現象**：`build-book.py:90` 只認 `^\[截圖:\s*(.+)\]$`（硬要半形冒號）。如果 md 裡是 `[截圖] xxx`、`[截圖：xxx]`（全形冒號）、`![截圖](path)`，或前後有多餘字符，全部走到 `<p>` 分支變普通段落。建議先 `grep -rnE '\[截圖' 01-教材內容/` 查出所有變體。

**修法**：
1. 放寬 regex：`^!?\[截圖[:：]?\s*(.*?)\s*\]` 並容許 `(...)` 後綴。
2. 原檔統一為 `[截圖：描述]`（全形冒號）。
3. 長遠：真的要出版就把截圖補上去（目前整本書 `<img>` = 0）。

### 5. Code block 內容用 `\`\`\`` 但源檔裡混雜了「沒有圍籬的偽代碼」
**位置**：第 5 講大量出現，其他章有零星跡象

**現象**：`Supabase.from('reservations').update(...)` 這類 JS 片段在某些段落**沒被 code block 包起來**，直接走到 `<p>` 分支後被 `html.escape` 變成 `&#x27;` `&amp;`，以 Noto Serif TC 字體呈現在正文裡，**非常難讀**。全書有 306 處 `&#x27; &amp; &quot; &lt; &gt;` 實體出現在 `<p>` 裡（正常應該只在 `<pre>` 裡）。

**修法**：審閱每章 md 原檔，把內嵌的代碼片段統一用 `` ` `` 或 ` ``` ` 包起。或在 `inline()` 對常見代碼字符做啟發式偵測（不推薦，容易誤傷）。

## 🟡 中度問題

### 6. 表格完全不支援
整本書 `<table>` / `<thead>` / `<tbody>` = **0**。原 md 如果有 `| 欄 | 欄 |` 表格全部會被逐行當 `<p>` 輸出（一堆管道符號垂直散落）。`md_to_html()` 沒寫 table 分支。

**修法**：要嘛在 build 腳本加 GFM 表格支援，要嘛直接改用 `markdown-it-py` / `mistune` 等成熟解析器換掉自刻 parser（強烈推薦，目前這個 parser 連有序清單、表格、腳註、引用區塊都不支援）。

### 7. `page-break-inside: avoid` 缺席 → 標題可能掛在頁尾
`.chapter { page-break-before: always }` 只管章層級。`h2 / h3 / h4` 沒 `page-break-inside: avoid` 也沒 `page-break-after: avoid`，**印刷時可能出現「標題孤雁」**（heading 在頁面最後一行，內容跳到下一頁）。`pre` 同理——一段 30 行的代碼塊可能硬切兩半。

**修法** CSS 補：
```css
@media print {
  h1, h2, h3, h4 { page-break-after: avoid; break-after: avoid; }
  h1, h2, h3, h4, li, pre, .screenshot-placeholder {
    page-break-inside: avoid; break-inside: avoid;
  }
}
```

### 8. 頁碼 counter 在 print 模式下邏輯錯誤
`book-preview.html:199–203`：print 時 `.page::after` 改為 `position: fixed; bottom: 1cm`。但 **`position: fixed` 在多頁列印下會每頁都印同一個數字**（通常是計數結果最後的值，不是當前實際頁碼）。目前依賴 CSS counter，但 `.page` 是每個 CSS box，多頁內容在 print 流式被 browser 分頁後 counter 行為不可預期。

**修法**：改用 CSS paged media 的 `@page { @bottom-right { content: counter(page); } }` 取得瀏覽器真實頁碼，把 `.page::after` 留作螢幕預覽用。

### 9. TOC 沒頁碼
目錄 `book-preview.html:223–228` 只有章名，**沒有「第 N 頁」**。出版書目錄必備。

**修法**：CSS paged media 有 `target-counter(attr(href url), page)` 可達成；但瀏覽器支援參差。Pragmatic 方案：先用 headless Chrome 列印一次生成 PDF，再 AST 回讀頁碼寫回 TOC。

### 10. 第 5 講的「`□` 勾選框清單」斷成碎片
位置 `book-preview.html:1712–1747`
現象：因為第 5 講的 code-block-bug（問題 1），`□ 填寫完整資料...` 這些應該是「測試清單」的勾選項，被夾在一堆 `<pre><code>` 片段中間，每個 `<p>` 之間還被空 code block 隔開。

**修法**：問題 1 修完後順帶檢視此區，改成 `<ul>` 或 `<ol>` 帶 `□` 符號的 task list 樣式。

### 11. `◆` 名詞符號在重要名詞區的呈現不一致
`build-book.py:101` 把 `◆` 歸類為 bullet 會進 `<ul><li>`，但如果前面是 `<p>` 段落緊接 `◆ xxx`，空行不正確時就走到 `<p>◆ xxx</p>` 分支（例如第 5 講 bug 區域）。出版時希望「重要名詞」每一條都是一個卡片式排版，建議獨立 class `.glossary-term`。

### 12. 中英文混排空白未規範
正文大量出現「AI 協作開發」「Supabase 資料庫」這類—有的有空白，有的沒有（如「AI 時代個人網站開發實戰」與「AI工具」混用）。建議用 [pangu.js](https://github.com/vinta/pangu.js) 或 `pangu-py` 在 build 階段全檔 normalize。

### 13. 章節間沒有「Part/單元分組」過渡
19 講按順序排，但實際上 1–4 是入門、5–6 是會員（上下）、12–14 是訂位/訂房類、18–19 是收官。**沒有 Part 標題頁或單元扉頁**，讀者感受不到節奏。

**修法**：在生成 HTML 時每 4–5 章插入一個 `.part-divider` 頁，形如「第一部分：入門四講」。

## 🟢 優化建議

### 14. 封面樣式偏單薄
`book-preview.html:57–92` 封面只有標題 + 副標 + 作者 + 年份 + 一條紅線。對一本書來說太素。建議加：
- 出版商 logo 或 thinker.cafe 識別
- ISBN / 初版日期
- 一段 2-3 行的封面引文（作者語或推薦語）
- 背景可加淡色紋理或咖啡漬 SVG（呼應「咖啡廳」的首講）

### 15. 封底缺失
目前結尾停在第 19 講 `</div></body>`，沒有封底頁、沒有作者簡介頁、沒有 Colophon（版權頁）。正式出版物應有。

### 16. 字體載入自 Google Fonts
`book-preview.html:9` `@import url('https://fonts.googleapis.com/...')`。**離線預覽 / 印刷 PDF 若無網路會 fallback 到 Times New Roman + Courier New**，視覺驟降。建議本地嵌入字體或用 base64。

### 17. 行距 `line-height: 1.8` 對中文書略鬆
中文書推薦 1.6–1.75。目前 1.8 在 12pt 字級下單頁容納字數會少 10% 左右，整本書可能多出 10+ 頁。對印刷成本有影響。建議試 1.7。

### 18. `pre` 在 print 用 `white-space: pre-wrap` 會強制折行代碼
`book-preview.html:206` 的 `pre { white-space: pre-wrap; word-wrap: break-word; }` 在 print 裡會把長代碼行硬斷。**JavaScript / SQL 代碼斷在任意字符處非常難讀**。建議改用 `white-space: pre` 配合 `overflow-x: hidden` + 視覺提示 `→` 表示截斷，或降低代碼字級到 8.5pt 讓它塞進 A4。

### 19. `.screenshot-placeholder` 佔位框過於樸素
目前只是灰底虛線框 + 中灰色文字，出版預覽裡看起來像遺漏。建議：
- 加尺寸提示（例如 `16:9 / 建議 1920x1080`）
- 編號（`[插圖 5-1]` 跟章節綁定）
- 給美術/攝影師當拍攝清單用

### 20. TOC 沒分區 / 沒縮排
19 個 `<li>` 平鋪。長度會超過一頁。建議分成 3-4 個 Part 段落，每部分內縮。

### 21. 第 7 講 / 第 15 講等使用破折號 `—`，第 8 講用 `-`
從 TOC 看：「第七講：法律事務所網站 — 專業形象…」vs「第八講:餐廳訂位網站 - 即時座位…」。這同時呼應問題 2，破折號全半形也要統一（全形破折號 U+2014 `—` × 2 才是標準中文用法 `——`，單個 `—` 也常見，但 `-` 半形減號在正式書裡絕對不行）。

### 22. 缺序言 / 前言 / 推薦序
正式出版物常見的「序」「自序」「推薦序」「如何使用本書」都沒有。至少應補一篇「給讀者的話」或「本書導讀」夾在 TOC 前。

### 23. 參考文獻格式不統一
第 5 講是 `1. **書名** - 作者：URL`，第 19 講是 `1. HTML5 UP 免費模板：https://...`，**沒有遵循 APA / Chicago 任一標準引用格式**。如果要正式出版（華視 / 鄒老師系列），需要全書統一 citation style。

### 24. 作者資訊過於簡略
封面只有「作者：鄒慶士、湯明軒」。沒有作者簡介（學經歷、著作、聯絡方式），也沒有貢獻聲明（誰寫哪幾講）。對於雙作者書是必要的。

### 25. 沒有 @page CSS 設定 A4
整個 CSS 用 `max-width: 210mm` 模擬 A4 寬度，但沒有 `@page { size: A4; margin: ... }`。列印時瀏覽器會用系統預設紙張，不一定是 A4。

**修法**：
```css
@page { size: A4; margin: 2.5cm 2cm; }
```

### 26. 「鄒老師系列」可能需要華視或學校的版權聲明頁
若預計交付華視或學術用途，應補版權頁（© 2026 作者姓名／出版社）、版次、ISBN placeholder、印刷規格說明。
