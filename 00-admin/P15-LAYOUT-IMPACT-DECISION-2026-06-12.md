## 3. 三案

### 案 A：純紙本標注（只上 L1+L2，放棄 L3）
龍虎零摩擦、7/3 三校時程最穩。但鄒老師電話的核心 ask（產出結果呈現層）不進書 — 等於電話白接，鄒老師三校還會卡同一點。**不推。**

### 案 B：紙本標注 + 定容整頁插入包（推薦）
- L1+L2+L4：照常標在二校紙本上（小節標題、圖表編號、句修都是紅筆等級；表 3-1 新表文與序言全文用附頁交付）。
- L3 改設計：每講的「AI 產出結果」**壓成定容 1 整頁圖版**（首頁截圖 + 產出清單 + 3 條驗收要點），以**獨立整頁、不進正文文字串**的形式插入第 5 節固定錨點。
- 關鍵：整頁圖版插入 = 既有文字框**零 reflow**，龍虎只需置入 18 頁 + 更新頁碼/目錄。這是教科書「圖版頁」常規作業，不是重排。
- 交付紀律：每講一個錨點、規格統一、**一次交清不迭代**（他們怕的是 churn，不是工作量）。圖檔 + 版面文字由我們備齊（300dpi PNG + 純文字）。
- 全書增量：+18 頁（章節雙頁起需求由龍虎按台數調整）。
- 前置：講 3 一頁版樣張先給鄒老師簽核 → Cruz 跟鄒老師通氣（老師手上正有紙本，避免他標出衝突的大改）→ 再對龍虎發包（framing：「三校時的固定插入頁，一次交清」）。

### 案 C：打掉重做（新版全書第二次重排）
內容端最乾淨（`01-current/` full pattern 直接出新 Word），但龍虎已靠邀過、再來一次傷關係，三校變成二校 redo，7/3 必炸。只有鄒老師判斷「產出結果層必須全幅 2 頁/講呈現」才值得，而他電話裡自己說過頁數有限、截圖+少量片段比大段 code 適合 — **與案 B 的定容設計一致。不推。**
