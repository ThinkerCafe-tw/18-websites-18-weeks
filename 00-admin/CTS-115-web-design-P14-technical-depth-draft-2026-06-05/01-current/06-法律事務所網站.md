# 第 06 講：法律事務所網站

本講任務：用 AI（Artificial Intelligence，人工智慧）協作完成一個法律事務所網站原型，讓使用者能理解案件類型、預約初談、查看律師資歷與服務邊界。

本講完成後，你會得到四個成果：

- 一份法律事務所網站需求摘要。
- 一個可展示的專業服務信任網站原型。
- 一份律師資歷、案件類型、初談預約、收費方式、免責聲明、聯絡資料的資料模型草稿。
- 一份手機版、信任證據、流程斷點與上線前檢查清單。


本講學習重點：

◆ 理解法律服務的本質：「焦慮的完全轉嫁」，不是法條的買賣
◆ 掌握律師事務所三種商業邏輯：案件篩選溢價、顧問費現金流、產值槓桿
◆ 用心理學解釋信任三支柱如何在高度資訊不對稱的情境裡發揮作用
◆ 認識三種讓當事人陷入更深痛苦的手法，理解為何不走這條路
◆ 學會把本章產業需求轉成頁面模組、資料模型、AI 提示詞與驗收清單

## 1. 破冰案例

**本章情境：週二上午十點，一份公文**


週二上午十點，新北市。

一個中年男人坐在自己辦公桌前，公文放在鍵盤旁邊。他已經看了這份東西三十分鐘，但每次讀到「臺灣臺北地方法院民事庭」那幾個字，他的眼神就會失焦。

公文的意思他大概懂，又好像不完全懂。他的公司和一家供應商有一筆合約糾紛，金額不大，一百五十萬。但這封傳票告訴他，對方已經決定不繼續協商，直接走訴訟了。

他打開 Google，打了「商務糾紛 律師 推薦」。

清單跳出來，他點進去第一家，是一間合夥律師事務所的官網。首頁是一張用 Photoshop 去背的律師坐姿照，背景是貼在一起的法典書背，光線是攝影棚的冷白燈，律師的表情是那種讓人想到「我現在要上台報告」的樣子。

他往下滑，看到業務範圍列表：商事法、家事法、行政法、智慧財產、刑事辯護、土地糾紛……二十一個項目，每個三個字，後面沒有任何說明。

他看著那二十一個項目，心裡升起一種說不清楚的感覺。

不是不信任。是那種在醫院掛號，護士問你「你哪裡不舒服」，你說「我脖子以下全部都不舒服」，然後她低著頭打了幾個字之後說「下一位」的那種感覺。

他關掉那個分頁。打開第二家。

第二家的網站首頁，第一行文字是：「一封陌生的法院傳票，對大多數人來說是人生中第一次。我們接過很多這樣的第一次。」

他停下來，往下繼續讀。

---

這個場景裡有一件事情值得停下來想：為什麼那一行字有效？

它沒有講專業、沒有講勝訴率、沒有列出律師學歷。它只做了一件事：讓那個男人感覺到「有人知道我現在的感受」。

這不是文案技巧。這是理解了那個男人在那一刻真正需要的是什麼。

**法律服務的客戶，買的不是法條。他買的是「把睡不著覺的夜晚交給另一個人」的權利。**

從這個理解出發，你為法律事務所做的每一個網站決策，才會有正確的出發點。

## 2. 業態需求

**產業聆聽：他在律師官網上到底在找什麼**


#### 三個核心恐懼

**恐懼一：費用黑洞。** 諮詢費、委任費、出庭費、裁判費、勝訴成功報酬金……加起來多少，沒有人在委任前算得清楚。最壞情境：打贏了，對方沒錢執行，自己卻先墊了一百萬律師費。

**恐懼二：專業傲慢的疏離感。** 很多人諮詢完說：「他一直講法條，我不知道那對我的情況代表什麼。」沒有翻譯成人話，當事人感覺自己是個不懂事的人被交代程序，而不是被照顧。

**恐懼三：敗訴後名譽崩解。** 那份判決書可能被 Google 永遠搜尋到。更深處還有一個問題：「如果輸了，是律師不夠努力，還是本來就不能贏？」這個疑問讓人無法完全信任任何律師。

#### 三個猶豫點

就算恐懼被初步排除了，還有三個讓人停在門口的猶豫點。

**第一：初次溝通，他在聽你說話，還是在評估你值不值得接？** 諮詢開頭幾分鐘，客戶很敏感地判斷對方是在「理解委屈」還是在「評估勝算」。有經驗的人五分鐘之內就感覺得出來。

**第二：勝訴率的誠實程度。** 說「找我就沒問題」的律師反而讓人警覺。真正的專家會告訴你難點在哪、不確定因素是什麼。「一定贏」三個字，讓人覺得對方要不是在說謊，要不是沒在認真評估。

**第三：專精還是全接？** 「什麼都接」和「我專做這個」，可信度差很大。帶著合約糾紛問全接律師，心裡會升起疑問：「他接過和我類似的案子嗎？」

#### 三個信任觸發

**常識化的法律說明。** 不是用法條解釋法律，是用故事解釋法律。「你的情況，核心問題是合約第十二條的爭議地帶，過去有三個類似案例……」這種說法，讓當事人覺得律師真的在處理他的問題，不是在套程序。

**透明的收費結構與進度回報。** 「費用分三個階段，你知道每個階段在付什麼，每兩週給你書面進度報告。」這直接解掉費用黑洞的恐懼，讓人覺得過程是被告知的，不是被蒙在鼓裡的。

**故事化的類似案例。** 不是勝訴率，是具體故事：「之前有個合約糾紛客戶，對方策略是拖，我們的反制是三件事……開庭前兩週和解，他拿回了應得的八成。」傳遞的不是「一定贏」，而是「我們知道這種情況怎麼應對」。

#### 人性真相

**客戶請律師不是為了聽法條，而是為了找一個人幫他承擔睡不著覺的夜晚。律師若能傳遞「我已接手，你可以去睡覺」，這種心理卸載的價值，遠超勝訴本身。**

### 2.1 產業營運邏輯

**產業心法：老手是怎麼算這門生意的**


#### 案件篩選能力與標的物溢價

老手不接所有案件。這不是傲慢，這是商業邏輯。

一件標的五千萬的土地糾紛，和一件標的五十萬的欠款糾紛，律師要讀的卷宗、要準備的狀書、要出庭的次數，差距沒有那麼大。但前者的委任費可能是後者的十倍。

老手把初次諮詢當成篩選工具，不是服務工具。大約八成的精力花在評估客戶的財務狀況、對案件的情緒化程度、以及這個案子能不能打出一個值得放在簡歷上的判決結果。剩下的兩成精力，才用來告訴客戶他的問題是什麼。

篩選得好，意思是：一年接二十個案子，每個案子的平均酬勞是接八十個普通案子的兩倍。時間投入少、獲利集中，這才是高年資律師的工時結構。

#### 常年法律顧問的穩定現金流

訴訟費是機會收入，顧問費是基礎收入。

懂得設計收入結構的律師，會把簽下常年法律顧問合約視為最優先的業務目標。對象是中小企業主，每月固定收費通常在一到兩萬元之間（實際金額因事務所規模與服務範圍而異）。服務內容多半是：審閱合約、諮詢電話、偶爾出一份法律意見書。實際耗費的時間，遠低於訴訟案件。

算一下：五十家中小企業顧問客戶，每月固定收入五十萬到一百萬，覆蓋辦公室租金和助理薪資。等這個數字穩定了，每一件訴訟委任收進來的酬勞，幾乎全部是淨利。毛利率在某些月份可以達到八到九成（業界觀察，各事務所結構不同）。

這個模型的關鍵洞察是：律師不是靠訴訟活著，律師靠顧問費活著，靠訴訟賺錢。

#### 合夥律師的產值槓桿

合夥律師的真實身份是接案者，而不是辦案者。

一個有經驗的合夥律師帶三名受僱律師或實習律師。判決書搜尋、書狀初稿、庭期安排、卷宗整理，這些工作由低薪助理完成。合夥律師負責的是：出庭、和解談判、客戶溝通、以及讓電話繼續響的名氣經營。

一名合夥律師的時間，只花在任何一名助理都無法替代的地方：法庭上的應變、客戶第一次諮詢時的感受管理、和對造律師喝咖啡時的資訊交換。

事必躬親的律師，體力到頂就是產值上限。懂得分工的律師，產值上限是整個團隊的總和減去薪資成本。

**老手算的從來不是一個案子能賺多少。他算的是一年有多少他親自處理的時間，每小時帶來的淨利是多少。**

## 3. 網頁設計重點

### 3.1 網站風格

法律事務所網站的風格要讓當事人感到專業、冷靜、可理解。過度華麗的動畫或侵略性標語會削弱信任，因為當事人通常是在焦慮中搜尋法律協助。

首頁應優先呈現服務領域、律師資歷、案件初談流程、費用區間與聯絡方式。設計重點不是把律師包裝成英雄，而是讓當事人知道下一步怎麼走。

### 3.2 建立信任

信任模組要避免勝訴保證。可呈現的是專長領域、辦案經驗類型、文章觀點、服務流程、費用計算方式與利益衝突檢查（Conflict Check）。

法律網站要清楚說明：送出表單不代表已成立委任關係，網站內容也不等於個案法律意見。這不是降低轉換，而是保護雙方。

### 3.3 引導行動

主要行動是填寫初談表單、預約諮詢、查看服務領域、閱讀費用說明。CTA（Call to Action，行動呼籲）應使用「預約初談」「提交案件初步資訊」這類中性文字，不應使用「保證勝訴」「立即翻盤」等誤導語句。

### 3.4 心理學錨點

**你的大腦在看見那個律師官網的那一刻已經在計分**


#### 信任三支柱

梅爾等人一九九五年在《管理學評論》（*Academy of Management Review*, 20(3), 709–734）提出信任模型：在資訊嚴重不對稱的情境下，信任取決於三件事——能力（他真的做得到嗎）、良善（他是為我想嗎）、誠信（他說的是真的嗎）。

法律服務幾乎是所有服務業裡資訊最不對稱的。客戶不懂法律，無法評估律師能力，無法驗證律師動機，在委任之前三個問題都沒有答案。但人不會因為沒答案就停止判斷，他們用**代用指標**填補空缺。

執業年資、得獎紀錄是能力的信號；案例故事裡有沒有「解決那個難點的敘事」，是良善的信號；收費透明與否、是否明說案子的困難，是誠信的信號。

**一個法律事務所的網站，本質上是一份信任三支柱的填答表。哪一欄空白，哪一欄就是客戶點關閉的原因。**

#### 權威效應與門面作為信號

席亞爾迪尼指出，人面對不熟悉的領域時，會把判斷力讓渡給看起來具備權威的人或環境。律師官網的那張深色西裝加厚重法典書牆，不是沒有原因的。

史彭斯（Spence）一九七三年在《經濟學季刊》（*Quarterly Journal of Economics*, 87(3), 355–374）提出信號理論：品質無法直接觀察時，投資高成本信號的行為本身就是品質的證明——低品質者無法負擔那個代價。精裝辦公室、整齊的執業資歷、律師公會認證，都是在說「我打算在這個行業待很久」。

**但這個邏輯有一個斷點：信號成本不夠高或可被複製時，它就失去品質指標的效力。** 一張股票照片貼上去，背後沒有那些執業年份，客戶的直覺仍然會偵測到哪裡不對，他說不清楚，但他不會委任。

### 3.5 信任底線

**三種讓當事人陷入更深痛苦的手法**


#### 底線一：刻意製造法律恐慌

諮詢時把對方一個微小行為講成「可能構成重罪」，利用當事人對法院的本能恐懼，在他最脆弱的時刻推他走進委任。這個手法借用信任三支柱裡「能力」那一欄：「我看到了你沒看到的風險，只有我能保護你。」

傷害是雙向的。對當事人，他打了一場可能從一開始就沒有必要的官司，激怒了對造讓事情更複雜。對整個行業，每一次這種操作都在削薄那層本來就很薄的信任。

#### 底線二：無效程序的重複計費

採用鐘點費計算的情況下，每一項程序都是計費單位。手法是建議當事人進行對案件結果沒有幫助的動作：明知超過時效的抗告還是提，明知對方資產查不到的假扣押還是送件。對當事人看起來是「律師在努力」，對律師是可以入帳的鐘點。

這個手法外觀上和「盡力為客戶爭取」幾乎一樣，當事人結束後只是隱約覺得「帳單有些地方很奇怪」，說不清楚、不會投訴，只是不再轉介、不再回頭。這種流失，是最沉默的商業傷害。

#### 底線三：包打官司與虛假勝訴承諾

「找我，這個案子一定贏。」

這句話違反了律師倫理規範（律師倫理規範第三十一條明定，律師不得對案件結果為不實之保證）。法律案件從來沒有「一定」，勝訴率取決於法官、對造策略、關鍵文件有沒有保存好。任何誠實的律師都知道這三個字說不出口。

輸了之後，責任被推給「法官沒有正確理解」「對造有手段」。當事人不只輸了官司，還多了一層被騙的感受，從此不信任任何一個律師。

---

**我們知道這些手法存在。我們選擇不用。你這輩子要做的不是「有效率的生意」，是「能在十年後還被信任的事務所」。**

## 4. 功能與資料模型

**實作主題：把「專業信任網站設計」落成可檢查的網站。**


前面幾層談的是產業、人性與風險；這一層要把那些判斷轉成網站結構。AI 協作不是請 AI 「做一個漂亮網站」，而是把業態需求、頁面模組、資料欄位、流程規則與驗收標準說清楚，再讓 AI 產出第一版。

### 本章網站需求

本講的網站型態是「專業服務信任網站」，核心行動是：理解案件類型、預約初談、查看律師資歷與服務邊界。

可以先用這張表整理需求：

| 模組 | 功能 | 必要內容 |
|---|---|---|
| Hero 首頁 | 建立第一印象 | 名稱、定位、主張、主要行動按鈕 |
| 信任證據 | 降低猶豫 | 真實照片、資歷、評論、規則或案例 |
| 核心流程 | 完成主要任務 | 理解案件類型、預約初談、查看律師資歷與服務邊界 |
| 資料表單 | 收集必要資訊 | 姓名、聯絡方式、情境欄位、備註 |
| 規則說明 | 避免誤解 | 價格、限制、取消或退款規則 |
| 聯絡與後續 | 接住下一步 | 電話、LINE、Email、地圖或確認訊息 |

### 頁面與功能模組

請把網站想成一組可檢查的模組，而不是一張長長的海報。每個模組都要回答一個使用者問題：我在哪裡？這適合我嗎？我相不相信？下一步要做什麼？

最小可行版本可以先做成單頁網站，但單頁不代表沒有結構。導覽列要能跳到主要區塊；主要行動按鈕要出現在首頁第一屏與流程結尾；表單送出後要有確認訊息，不能讓使用者不知道資料是否送出。

### 資料與流程

這一章至少要先拆出下列資料物件。第一版不一定要真的接資料庫，但你必須知道網站背後需要哪些資料，AI 才不會只產出表面文案。

| 資料物件 | 欄位 | 範例或用途 |
|---|---|---|
| LawFirmProfile | 名稱、地區、定位、營業或服務時間 | 用於頁面內容、表單或狀態檢查 |
| UserRequest | 姓名、聯絡方式、需求、備註 | 用於頁面內容、表單或狀態檢查 |
| ServiceItem | 名稱、內容、價格或條件、限制 | 用於頁面內容、表單或狀態檢查 |
| ScheduleSlot | 日期、時段、名額、狀態 | 用於頁面內容、表單或狀態檢查 |
| Policy | 取消、退款、隱私或服務邊界 | 用於頁面內容、表單或狀態檢查 |
| TrustEvidence | 照片、評論、資歷、案例或證明 | 用於頁面內容、表單或狀態檢查 |

流程上可以用這條線檢查：

```text
看見首頁 → 判斷是否適合 → 查看信任證據 → 進入核心流程 → 填寫必要資料 → 看到確認或下一步
```

如果中間任何一步需要人用 LINE 或電話人工補洞，就要在網站上說清楚。不要讓網站假裝已經完成系統能力，實際上卻只是收一張表單。

### 案件初篩與利益衝突檢查

法律事務所網站不能只放律師照片與服務項目。真正專業的網站，應該先把「能不能接這個案子」和「適不適合由本所處理」說清楚。

初談表單不是越詳細越好。它要收足以初步判斷案件類型的資料，但不能讓當事人在尚未建立委任關係前，以為自己已經取得法律意見。

可以把資料模型拆成：

| 資料物件 | 重要欄位 | 用途 |
|---|---|---|
| `PracticeArea` | 案件類型、服務範圍、常見不承接情況 | 說明本所能處理什麼 |
| `AttorneyProfile` | 律師姓名、資格、專長、服務語言 | 建立專業信任 |
| `IntakeRequest` | 聯絡方式、案件類型、對造姓名、簡短事實 | 初步案件判斷 |
| `ConflictCheck` | 當事人、對造、關係人、檢查狀態 | 利益衝突檢查 |
| `ConsultationBooking` | 初談日期、方式、費用、狀態 | 安排初談 |
| `EngagementBoundary` | 是否已委任、服務範圍、費用計算 | 明確委任邊界 |

案件初篩可以用這條線理解：

```text
送出初談表單 → 利益衝突檢查 → 確認可初談 → 初談 → 決定是否委任
                                      │
                                      └→ 不適合 / 轉介 / 不承接
```

其中「利益衝突檢查」很重要。法律服務不像一般預約，律師不能同時服務利益衝突的雙方。表單中至少要詢問對造或相關人名稱，並在文案中說明：

```text
送出表單不代表本所已接受委任。本所需完成利益衝突檢查與初步確認後，才會安排正式初談或委任。
```

這句話不是冷漠，而是專業邊界。

### 免責聲明與委任邊界

法律網站最容易出問題的地方，是把一般法律資訊寫得像個案法律意見。

比較安全的資訊呈現方式：

| 類型 | 可以寫 | 不應該寫 |
|---|---|---|
| 法律文章 | 一般流程、常見問題、應準備文件 | 直接判斷讀者個案輸贏 |
| 案例介紹 | 去識別化、說明爭點與處理方向 | 暗示相同結果可複製 |
| 費用說明 | 初談費、計時費、固定費可能範圍 | 保證低價或保證勝訴 |
| 表單回覆 | 安排初談、確認資料 | 直接給法律結論 |

網站應該清楚標示三個邊界：

- 閱讀網站內容不形成律師與當事人關係。
- 送出表單不代表事務所已接受委任。
- 個案法律意見需在律師了解完整事實後才能提供。

法律事務所網站的信任，不是把話說滿，而是把邊界說清楚。

## 5. AI 協作實作

可以用下面這份提示詞作為第一版：

```text
你是網頁設計老師與前端工程師。
請幫我製作一個「法律事務所網站」的單頁網站原型。

【網站目標】
律師簡介、案件類型、初談流程、收費說明與表單

【使用者情境】
使用者第一次進入網站時，需要在手機上快速判斷這個服務是否適合自己，並能完成主要行動：理解案件類型、預約初談、查看律師資歷與服務邊界。

【必要區塊】
1. Hero 首頁：名稱、定位、主張、主要行動按鈕
2. 信任證據：真實照片、資歷、評論、規則或案例
3. 核心流程：理解案件類型、預約初談、查看律師資歷與服務邊界
4. 資料表單：姓名、聯絡方式、情境欄位、備註
5. 規則說明：價格、限制、取消或退款規則
6. 聯絡與後續：電話、LINE、Email、地圖或確認訊息

【資料欄位】
請在頁面中示範下列資料結構：
1. LawFirmProfile：名稱、地區、定位、營業或服務時間
2. UserRequest：姓名、聯絡方式、需求、備註
3. ServiceItem：名稱、內容、價格或條件、限制
4. ScheduleSlot：日期、時段、名額、狀態
5. Policy：取消、退款、隱私或服務邊界
6. TrustEvidence：照片、評論、資歷、案例或證明

【設計要求】
1. 請使用清楚的 HTML（HyperText Markup Language，超文字標記語言）結構與 CSS（Cascading Style Sheets，階層式樣式表），做成單一 index.html。
2. 請使用 RWD（Responsive Web Design，響應式網頁設計），手機、平板、桌機都要可讀。
3. 請避免假評價、假資歷、假價格或過度承諾。
4. 表單與按鈕要有清楚的互動狀態，可以用靜態示意，不需要真的送出到伺服器。
5. 請直接輸出完整程式碼，不要解釋。
```

## 6. 上線與延伸

AI 產出後，不要只看第一眼漂不漂亮。請依序檢查：

- 首頁第一屏是否說清楚服務適合誰與下一步是什麼。
- 資料欄位是否足以支撐核心流程，而不是只留下姓名電話。
- 初談表單是否收集案件類型、對造或關係人，支撐利益衝突檢查。
- 是否明確說明送出表單不等於建立委任關係。
- 法律文章是否避免直接判斷個案輸贏或保證結果。
- 費用說明是否有範圍與計算方式，而不是只寫「歡迎洽詢」。
- 價格、限制、取消或隱私規則是否清楚。
- 手機版是否能完成主要行動，且文字與按鈕不重疊。

如果要上線，可以先用 GitHub（程式碼與專案託管平台）保存檔案，再用 Vercel（前端部署平台）或 GitHub Pages 部署。這裡的 GitHub 不是工程師炫技，而是讓每次修改都有紀錄；部署也不是最後才做的事，而是讓你用真實手機連結檢查網站是否可用。

本講最重要的練習，是把「產業洞察」翻譯成「可檢查的網站需求」。當你能說清楚資料、流程、信任與風險，AI 才會從模板生成器，變成真正的協作助理。

### 6.1 重要名詞

**常年法律顧問（Legal Retainer）**：企業與律師事務所簽訂的長期服務合約，固定月費換取合約審閱、法律諮詢等服務。實際耗時低，是事務所穩定現金流的基礎收入。

**信任三支柱**：梅爾等人（Mayer et al., 1995）信任模型的三要素：能力（Ability）、良善（Benevolence）、誠信（Integrity）。任一缺失，委任關係就難以成立。

**信號理論（Signaling Theory）**：史彭斯（Spence, 1973）提出，品質無法直接觀察時，投資高成本信號的行為本身就是品質的證明，因為低品質者無法負擔那個代價。

**權威效應（Authority Effect）**：席亞爾迪尼定義的影響力原則之一，指人面對不熟悉的領域時，傾向信任看起來具備專業權威的對象。

**假扣押（Provisional Seizure）**：訴訟前防止對造轉移財產的保全程序。是否有法律依據和實際執行效益，需謹慎評估。

**成功報酬金（Contingency Fee）**：案件勝訴或和解後，律師依約定比例額外收取的酬金。計算基準與觸發條件應在委任之初明確約定。

**案件初篩**：律師事務所在正式初談或委任前，先判斷案件類型、急迫性、是否屬於本所服務範圍的流程。

**利益衝突檢查**：確認事務所是否曾服務對造或相關利害關係人，避免同一律師或事務所同時處理利益衝突案件。

**委任邊界**：明確說明何時開始建立律師與當事人關係、服務包含哪些事項、費用如何計算。邊界越清楚，越能降低後續爭議。

### 6.2 自我評量

**第一題（是非）**：法律服務的客戶，委任律師的主要動機是獲得法律條文的正確解釋。

**第二題（是非）**：常年法律顧問合約毛利高，主要因為這類服務的實際時間投入通常遠低於訴訟案件。

**第三題（選擇）**：梅爾等人（1995）的信任模型中，信任的三個核心要素是：
（甲）資歷、勝訴率、媒體曝光
（乙）能力、良善、誠信
（丙）外表、年資、辦公室環境
（丁）學歷、知名度、客戶數量

**第四題（情境）**：一位企業主登入事務所官網，看到一張西裝律師加法典書牆的照片，以及二十一個服務項目清單，沒有任何案例或費用資訊。根據信任三支柱，這個網站在哪兩欄是空白的？這兩個空白如何影響委任決策？

**第五題（選擇）**：律師在諮詢過程中說「你放心，找我一定贏」，這句話最直接違反了哪一個原則？
（甲）收費透明原則
（乙）律師倫理規範中禁止對案件結果做不實保證的規定
（丙）信號理論中高成本信號的可信度要求
（丁）案件篩選的商業邏輯

**第六題（是非）**：根據本講產業營運邏輯，合夥律師的主要商業角色是親自完成所有案件的法律研究與書狀撰寫，以確保品質。

**第七題（情境）**：一位律師建議提出他明知超過時效的抗告，並收取了鐘點費。從本講信任底線 的角度，說明這對當事人、事務所信譽、整個法律服務業各造成什麼傷害。
