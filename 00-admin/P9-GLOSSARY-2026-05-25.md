# P9 — 全書專有名詞 Glossary

日期：2026-05-25

用途：統一全書第一次出現專有名詞時的中英文說法，避免授課老師或學生在不同章節看到不一致翻譯。

## 網頁與部署

| 縮寫 / 名詞 | 建議第一次寫法 | 白話說明 |
|---|---|---|
| AI | AI（Artificial Intelligence，人工智慧） | 用於協助產出、修改與檢查網站 |
| RWD | RWD（Responsive Web Design，響應式網頁設計） | 讓網站在手機、平板、桌機都能閱讀與操作 |
| SEO | SEO（Search Engine Optimization，搜尋引擎最佳化） | 讓搜尋引擎更容易理解網站內容 |
| CTA | CTA（Call to Action，行動呼籲） | 引導使用者下一步，例如預約、詢問、查看方案 |
| HTML | HTML（HyperText Markup Language，超文字標記語言） | 網頁內容結構 |
| CSS | CSS（Cascading Style Sheets，階層式樣式表） | 網頁視覺樣式 |
| JavaScript | JavaScript（網頁互動程式語言） | 處理按鈕、表單、狀態切換等互動 |
| Git | Git（版本控制工具） | 記錄每一次修改 |
| GitHub | GitHub（程式碼與專案託管平台） | 保存網站檔案與修改紀錄 |
| Vercel | Vercel（前端部署平台） | 將網站發布成公開網址 |
| DNS | DNS（Domain Name System，網域名稱系統） | 將網域指向網站伺服器 |

## 功能型網站

| 名詞 | 建議第一次寫法 | 白話說明 |
|---|---|---|
| Data Model | 資料模型（Data Model） | 把網站需要管理的資料拆成物件與欄位 |
| State Machine | 狀態機（State Machine） | 列出可訂、保留中、已付款等狀態與轉換規則 |
| Webhook | Webhook（外部服務事件通知） | 金流或外部服務主動通知網站付款成功等事件 |
| API | API（Application Programming Interface，應用程式介面） | 系統之間交換資料的接口 |
| CRM | CRM（Customer Relationship Management，客戶關係管理） | 管理客戶資料、紀錄與回訪提醒 |
| LTV | LTV（Lifetime Value，客戶終身價值） | 一位客戶在長期關係中可能帶來的總收入 |
| Ledger | 流水帳（Ledger） | 記錄點數、儲值、課程堂數等增減紀錄 |
| Waitlist | 候補名單（Waitlist） | 名額滿後依順位通知下一位 |
| Concurrency Conflict | 併發衝突（Concurrency Conflict） | 兩人同時搶同一房、同一時段或同一名額 |

| 名詞 | 建議第一次寫法 | 白話說明 |
|---|---|---|

## 高敏感領域

| 名詞 | 建議第一次寫法 | 白話說明 |
|---|---|---|
| Data Minimization | 資料最小化（Data Minimization） | 只收完成服務必要的資料 |
| Duty of Confidentiality | 保密義務（Duty of Confidentiality） | 諮商或專業服務中對個案資訊的保密責任 |
| Conflict Check | 利益衝突檢查（Conflict Check） | 確認律師或事務所是否能承接案件 |
| Informed Consent | 知情同意（Informed Consent） | 使用者了解服務內容、風險與資料用途後同意 |

## 寫作規則

1. 第一次出現縮寫時，建議用「英文縮寫（英文全名，中文翻譯）」。
2. 同一章後續可直接使用縮寫，不必每次重複全名。
3. 醫療、法律、諮商相關名詞，應維持「網站資訊呈現」角度，不寫成診斷、法律意見或心理治療建議。
4. 若名詞對學生不是必要，可放在重要名詞區，不要塞進正文第一段。
