# P15c 字數合規精煉規格（chapter agent 必讀）

背景：空院規格「每講 5,000 字為宜、至多不超過 7,000 字」，超標會被學校退稿刪減。現稿 17/18 講破頂。
鄒老師同時要求「不能淺碟、要有深度」。所以這是**精煉，不是抽筋** —— 砍字不砍知識點。

## 目標

- 每講純中文字（CJK）壓到 **6,000–6,800**（靶心 6,300，硬頂 6,800）。
- 量字方法：`python3 -c "import re;print(len(re.findall(r'[一-鿿]', open('FILE').read())))"`

## 絕對不動（動了就是破壞 P15 成果）

1. 所有 GFM 表格（含「表 N-X」題名行）——鄒老師要的編號對象。
2. 所有圖：5.2 的截圖 pointer 行、圖題行、流程圖 code block + 圖題。
3. §2 結尾的「因此本講的技術主題是「X」。」定位句。
4. `### 5.2 AI 產出結果` 整節（截圖、產出清單、結構骨架、media query 節錄、業態對照）。
5. §5.1 的提示詞 code block 本體 + 其後的「不要解釋」釐清段。
6. `### 6.3 參考文獻` 全部條目。
7. §1 破冰案例的**核心故事與人物**（可精煉文字，不可刪掉故事）。

## 精煉手法（砍這些）

- 贅字虛詞：「其實」「事實上」「換句話說」「也就是說」「我們可以發現」等填充語。
- 重複論點：同一個道理在 §2/§3/§4 講了三遍 → 留最有力的一處，其餘刪。
- 過度舉例：一個論點配三個例子 → 留最貼業態的一個。
- 展開式 → 要點式：把「鋪陳半段才講重點」的段落，改成直接給重點。
- 跨節重複鋪陳：§3 設計重點與 §4 資料模型若有重疊敘述，合併。
- 收束清單前的長導言：bullet list 前的三句鋪陳 → 一句。

## 保深度（這些是「知識點」，字可精煉、資訊不可少）

- 行業 know-how、為什麼這樣設計的因果、技術概念、資料模型欄位、狀態流程、驗收清單項目。
- 精煉後每講仍要看得出「這是懂這行的人寫的」，不是通用模板。

## 自檢（回報前必跑）

1. CJK 字數 ≤ 6,800（回報實際數字）。
2. 表格數、圖數、技術主題句、5.2 標題、6.3 標題 grep 確認都還在（回報各項計數，與精煉前一致）。
3. 通讀一遍：語氣連貫無跳 tone、無因刪段造成的指代斷裂（如「如前所述」指向已刪段落）。
4. 抽 3 個技術知識點，確認精煉後仍在。
