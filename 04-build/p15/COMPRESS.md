# P16 字數收斂規格書（chapter agent 必讀）

背景：空院規格**每講 5,000 字為宜、至多不超過 7,000 純中文字**，超標會被退稿刪減。現稿 17/18 講破頂。目標：把每講收斂到 **6,300–6,800 字**（硬頂 6,800，下限 5,000），同時保留 P15 的全部價值。**圖不計字數 — 36 張截圖全部保留，這本書要又瘦又圖文並茂。**

字數定義：純中文字（CJK 字元），自檢指令：
`python3 -c "import re,sys;print(len(re.findall(r'[一-鿿]',open(sys.argv[1]).read())))" <file>`

## 必保留（動了就是事故）

1. 技術主題定位句（§2 結尾「因此本講的技術主題是「X」。…」）
2. 全部 36 張圖的圖題行與（圖檔：…）pointer 行
3. 5.1 提示詞 code block 全文 + prompt 指令釐清段
4. 表/圖編號體系：刪表格後必須**重排連續**並同步改正文引用，禁出現懸空的「如表 N-X」
5. 6.1 檢查清單（可壓縮）、6.2 自我評量（不可刪題）、6.3 參考文獻（一條都不能少）
6. 骨架公式完整性：業態-[競爭優勢-網頁設計-功能與資料]-[AI部署與應用]-上線與延伸
7. 教材腔與銜接：刪減處要補銜接句，通讀不可有斷裂感

## 砍什麼（依優先序，砍下來的內容必須 PARK，不是丟掉）

1. **5.2 壓成定容版**（最大刀）：保留「貼給 AI→存檔→圖 N-a」開頭、兩張圖、產出清單壓到 3-4 bullets、業態對照 1 句、「結果不會一模一樣」1 句。**刪**：HTML 骨架示意 code block、media query 逐字節錄、冗長首屏描述 → park 到 website 檔。目標 5.2 文字 ≤500 字。
2. **心理學/行為經濟學長論證**：每段壓到 2-3 句（學者+年份+結論一句+應用一句），完整論證 park 到 hanshou 檔。引用照舊指向 6.3。
3. **重複性檢查清單與長條列**：同概念出現兩次的，留一次；十條的清單挑五條最有教學價值的。
4. **深度技術段**（webhook 細節、sandbox/production 長段、狀態機展開敘述）：正文留概念與一張表/圖，操作細節 park 到 hanshou 檔。
5. **破冰案例**：超過 600 字的壓到 400-500 字，保留情緒鉤子。
