# 第 05 講溢出素材 — 配套網站候選

> 來源：第5講 §5.2 AI 產出結果（首屏完整描述與圖檔替換註解）

首頁第一屏包含店名、一句話主張（其中直接寫出「療程中不進行銷售對話」），以及「會員入口」「立即預約」兩個按鈕；往下依序是服務項目、會員註冊、會員儀表板、線上預約、點數與儲值規則、消費紀錄與資料模型示意。照片先以免費圖庫的示意圖代替，AI 也用註解標出了日後要替換的正式檔名，例如 `images/service-clean.jpg`。

`header` 是店名與六個區塊的導覽連結；`footer` 是店名與地址示意、Email、營業時間與版權聲明。

> 來源：第5講 §5.2 AI 產出結果（media query 節錄）

手機版的關鍵是 CSS 裡的 media query。AI 產出的規則節錄如下：

```css
/* 平板：服務卡、規則卡與儀表板數字改為兩欄 */
@media (max-width: 1024px) {
  .grid, .stats { grid-template-columns: repeat(2, 1fr); }
  .hero .inner { grid-template-columns: 1fr; }
}
/* 手機：全部單欄，header 改為直向，按鈕與表單欄位整排可點 */
@media (max-width: 640px) {
  .grid, .cards, .stats { grid-template-columns: 1fr; }
  header .inner { flex-direction: column; align-items: flex-start; gap: 6px; }
  .btn { display: block; width: 100%; text-align: center; margin: 0 0 10px; }
  h1 { font-size: 1.6rem; }
}
```

用手機寬度打開同一個檔案，會員儀表板的等級、點數、儲值餘額會改成單欄一張一張往下排，按鈕與表單欄位也會放大成整排可點。

> 來源：第5講 §2.1 產業營運邏輯（預付卡的「沉澱資金」與展店槓桿，量化估算）

儲值卡作為低成本資金借貸工具的展店槓桿，可以用一個量化例子看清楚。客人儲值五萬送一萬，看起來是促銷，但老闆看到的是：四萬九千元現金先進了帳戶，而她的實際消費可能要分散在六個月內慢慢花完。這六個月裡，那筆預收現金可以拿去支付原料採購與美容師薪資，甚至作為第二家分店的頭期款 — 等於客人提前借了一筆無息週轉金給店家。

放大到全店規模：一家有三百位活躍會員的美容院，一季儲值回收金可能落在三百萬到五百萬元之間。這是一筆零利率的資金，比跟銀行借錢便宜得多，也是預付制美容院敢用「儲值送大額」當主打的真正原因 — 折扣不是讓利，是用一點帳面成本換一整季的現金流與展店本錢。

> 來源：第5講 §2.1 產業營運邏輯（師徒制的人力成本控制，職等分工細節）

美容院的人力結構通常是嚴格的職等制，這套分工是邊際利潤的隱形支柱。剛入職的「助理」負責洗臉、卸妝、備料、清潔療程床 — 這些是服務流程中最耗時、但技術含量較低的環節，因此用薪資較低的人力承擔；「主任」或「資深美容師」則只在需要技術判斷和銷售對話的關鍵時刻出現。

這個結構的設計意圖很清楚：讓「時薪最貴的人」只把時間花在「價值最高的工作」上，不被洗備清潔這類低價值工序消耗。整體人力成本因此能控制在營收三成以內，本就偏薄的邊際利潤才維持得住。換句話說，老手算的從來不是單一療程的服務費，而是這個小時裡哪些工作可以交給成本最低的人做、高技術人員的時間有沒有被用在值得的地方。
