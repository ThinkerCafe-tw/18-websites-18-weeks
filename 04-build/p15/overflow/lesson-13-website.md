# 第 13 講 overflow — 配套網站候選素材

> 來源：第13講 §4

## 基礎資料物件表（原表 13-2，與 5.1 提示詞【資料欄位】對應）

本講至少要先拆出下列資料物件。第一版不一定要真的接資料庫，但你必須知道網站背後需要哪些資料，AI 才不會只產出表面文案。

| 資料物件 | 欄位 | 範例或用途 |
|---|---|---|
| `AutoServiceProfile` | 名稱、地區、定位、營業或服務時間 | 用於頁面內容、表單或狀態檢查 |
| `UserRequest` | 姓名、聯絡方式、需求、備註 | 用於頁面內容、表單或狀態檢查 |
| `ServiceItem` | 名稱、內容、價格或條件、限制 | 用於頁面內容、表單或狀態檢查 |
| `ScheduleSlot` | 日期、時段、名額、狀態 | 用於頁面內容、表單或狀態檢查 |
| `Policy` | 取消、退款、隱私或服務邊界 | 用於頁面內容、表單或狀態檢查 |
| `TrustEvidence` | 照片、評論、資歷、案例或證明 | 用於頁面內容、表單或狀態檢查 |

> 來源：第13講 §5.2

## AI 產出首頁的首屏內容描述（圖 13-2 對照文）

首頁第一屏包含店名、地區與定位、一句話主張「先報價、後動工」，以及「查詢服務項目」「預約保養」兩個按鈕，按鈕下方標出營業時間；往下依序是信任證據、服務項目、預約保養、車籍紀錄與下次提醒、規則說明、聯絡與後續。廠內照片與地圖先以免費圖庫的示意圖代替，AI 也用註解標出了日後要替換的正式檔名，例如 `images/garage-01.jpg`。

值得注意的是，信任證據區塊放的是「先報價、後動工」「施工照片即時回傳」「看得懂的紀錄」三條做事規則與廠內照片示意圖，沒有出現編造的客戶評價與資歷，這是提示詞設計要求第 3 點在發揮作用。

> 來源：第13講 §5.2

## HTML 骨架示意

整體結構可以用下面的骨架示意（省略號代表實際內容，不是可直接貼用的程式碼），這就是第 13 講表 13-1 的需求模組落到程式碼後的樣子。其中表 13-1 的「核心流程」被 AI 拆成服務項目、預約保養、車籍與提醒三個 `section`，是常見而可接受的拆法：

```html
<header>…店名與導覽…</header>
<main>
  <section class="hero">…名稱、定位、主張、查詢服務項目、預約保養…</section>
  <section id="trust">…三條透明守則與廠內照片示意圖…</section>
  <section id="services">…服務項目、參考價格、所需時間、限制…</section>
  <section id="booking">…本週時段示意與預約表單…</section>
  <section id="records">…保養紀錄示意、提醒類型與提醒範例…</section>
  <section id="policy">…價格、取消、退款與個資規則…</section>
  <section id="contact">…電話、LINE、Email、地圖與後續說明…</section>
</main>
<footer>…店名、營業時間、版權…</footer>
```

> 來源：第13講 §5.2

## 響應式 media query 節錄

手機版的關鍵是 CSS 裡的 media query。AI 產出的規則節錄如下：

```css
/* 平板：服務與信任卡片改為兩欄 */
@media (max-width: 1024px) {
  .grid { grid-template-columns: repeat(2, 1fr); }
}
/* 手機：全部單欄，header 改為直向排列，按鈕與表單欄位維持易點 */
@media (max-width: 640px) {
  header { flex-direction: column; align-items: flex-start; gap: 0.4rem; }
  .grid { grid-template-columns: 1fr; }
}
```

用手機寬度打開同一個檔案，服務項目會改成單欄卡片，預約表單欄位也會整排可點。
