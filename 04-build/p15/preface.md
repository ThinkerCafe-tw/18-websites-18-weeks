# 序

教過網頁設計的老師，多半遇過同一個場景：第一週講 HTML 標籤，第二週講 CSS 選擇器，第六週學生終於做出一個置中的方塊，卻說不出自己為什麼要做網站。語法進度走完了，網站還是沒有出現。期末交出來的作品，往往是一頁套用範本的自我介紹，與真實世界裡任何一個會被搜尋、會被詢價、會被客人拿來做決定的網站，距離很遠。這不是老師教得不好，也不是學生不用功，而是課程的起點放錯了位置。這本書，就是從重新安排這個起點開始的。

本書的編排與多數網頁設計教材不同。傳統教材從語法開始：先學標籤、再學樣式、再學版面配置，等基礎打完，才回頭談「做一個網站」。本書反過來，從「網站要解決什麼問題」開始：一位攝影師的作品明明不差，為什麼客人都流向有網站的同行；一間診所為什麼需要線上預約，才能把電話從櫃檯解放出來；一個活動報名頁，要在幾秒內讓訪客判斷值不值得留下資料。全書十八講對應十七種業態與一次總複習，每一講都先把這個業態的客人、需求、信任條件與營運邏輯講清楚，再透過 AI 協作把網站實際做出來，最後回頭檢視頁面結構、資料流程、部署方式與驗收清單。

必須先說清楚的是：這樣的安排不是取消 HTML 與 CSS。語法仍然在書裡，每一講的產出仍然要打開來讀、要看得懂、要能動手改。改變的只是學習的起點——從手刻語法，移到業態需求與 AI 協作。過去學生必須先寫得出每一行程式，才看得到一個完整的網站；現在 AI 可以先把網站做出來，學生從一個會動的成品開始，往回追問每個區塊為什麼存在、每段樣式在控制什麼、哪裡寫得不符合需求。理解的路徑變短了，但理解的要求並沒有降低——看不懂結構的人，連 AI 做錯了都發現不了。

學生要學會的東西，其實一樣多。頁面結構怎麼安排，才能讓訪客在第一屏就知道這個網站賣什麼；資料怎麼流動，表單送出後去了哪裡、預約紀錄存在哪裡；網站怎麼從本機的一個檔案，經過版本控制推到正式環境；上線之後，要拿什麼清單檢查手機版面、搜尋設定與聯絡動線。差別只在於這些能力不再以背語法為第一步，而是在一個又一個真實業態的專案裡，被需求逼著長出來。會默寫 media query 的語法，不代表懂響應式設計；知道客人會在捷運上用手機看你的價目表，才是真的懂。

這樣的轉變對面授老師意味著什麼，是兩位作者在編寫過程中反覆討論的問題。我們的答案不是「老師被取代」，而是老師的角色往上移了一層。當語法講解可以交給教材與 AI 分擔，課堂上空出來的時間，正好用在 AI 做不好、也最需要經驗的三件事：第一是設計判斷——學生的網站好不好看是其次，符不符合這個業態的商業邏輯才是關鍵，這需要老師帶著學生對照真實案例去看；第二是需求翻譯——學生常常說得出「想要好看一點」，卻說不出好看是指留白、層級還是配色，把模糊的想法翻譯成 AI 聽得懂的明確指令，本身就是這門課最核心的訓練；第三是驗收——成品能不能真的上線營運，要靠人逐項檢查，而不是看畫面順眼就放行。這三件事過去在課堂上最缺時間做，現在終於有時間做了。

教學上，本書可以照講次順序使用，前三講建立從需求到上線的完整流程，之後各講逐步加入表單、預約、會員、金流對接等主題；也可以依學生背景挑選業態，讓開咖啡廳的學生直接從咖啡廳那一講進場。每一講的結構固定：破冰案例、業態需求、網頁設計重點、功能與資料模型、AI 協作實作、上線與延伸，老師備課時可以直接對應到一週的課堂節奏，學生自學時也能按步驟自行完成。

網頁設計教育的目標，從來不是把每個學生都訓練成前端工程師，而是讓他們有能力把一個真實的需求，變成一個能被找到、被信任、被持續經營的網站。工具換了一代，這個目標沒有變。我們希望這本書能讓老師站在更高的位置教學，讓學生在第一週就摸到真實的問題，也讓「會做網站」這件事，重新回到它原本該有的意義：解決某個人的某個問題。
