# P15 批次施工規格書（chapter agent 必讀）

目標：把 6/11 鄒老師電話的 P15 補橋 pattern 套到本講，**今晚出完整電子書給鄒老師校稿**。
參考範本：`01-current/03-攝影師作品集.md`（已完成 pattern + 過 3 路對抗驗證）。照它的做法，不要自創格式。

## 鐵律

- 手術式修補：只加不刪，不重寫既有正文，不改作者語氣（繁中教材腔；禁 AI 助手腔、禁「我們已經完成」式施工語）。
- 不碰其他章的檔案；不 git commit。
- 編號各章獨立（表 N-X / 圖 N-X，N=講次），按出現順序，不跨章連號。
- 任何 URL 必須先 `curl -s -o /dev/null -w "%{http_code}" -L <url>` 驗證 200 才能寫進參考文獻。禁止憑記憶編 URL。
- DOI 類 URL（JSTOR/AOM 等出版社常對 curl 回 403）：以 `curl -s -o /dev/null -w "%{http_code}" https://doi.org/api/handles/<DOI>` 回 200 視同通過。
- 教材中的範例網域一律用保留網域（`.example`、`example.com`），不得出現真實可註冊網域。

## 施工項（依序）

1. **表格編號**：每個 GFM 表格上方加粗體題名行 `**表 N-X　<題名>**`（全形空格 U+3000），題名行與表格間空一行；前文引入句改為「整理如表 N-X」「如表 N-X 所示」等自然句式。若引入句宣告的數量與表格列數不符，修正到一致（參考講 3 表 3-1 的修法）。
2. **第 5 節重構**：拆成 `### 5.1 本講提示詞`（既有 prompt）→ `### 5.2 AI 產出結果`（全新，見下）→ `### 5.3 結果檢查與修改`（既有檢查內容）→ 若本講有保存/部署/上線操作段落則 `### 5.4 <依內容命名>`。**講 11 已有自己的 5.x 小節：整合插入、順位重編，不得刪既有內容。**
3. **Prompt 指令釐清**：在 prompt code block 之後加一段，**逐字引用**該章 prompt 的收尾指令（多數章寫「不要解釋」——照原句引），說明這是給 AI 的輸出格式限制（目的：拿到可直接存檔的完整程式碼），不是教材不解釋。參考講 3 的段落寫法。
4. **5.2 AI 產出結果**（核心新增，照講 3 的 5.2 結構）：
   - 開頭：存成 `index.html` 用瀏覽器打開會得到類似圖 N-X 的首頁。（圖題編號一律用「圖 N-X」、接續本講既有圖號按第 7 項統一順序排；檔名固定用第 6 項的 `fig-NN-0X` 連號規格。圖題編號與檔名序號不必相同，排版配圖以 pointer 行為準。）
   - `**圖 N-X　AI 依本講提示詞產出的＜業態＞網站首頁（桌機版）**` + 下一行 `（圖檔：`fig-NN-0X-home-desktop.png`，隨稿附件）`。
   - 首屏與區塊描述段（依實際產出寫，不得描述不存在的東西）。
   - 產出清單 bullets（檔案、header、main 內各 section、footer）。
   - 結構骨架示意 code block（明寫「省略號代表實際內容，不是可直接貼用的程式碼」），並對照本章區塊表格（如有），不一致處補半句說明。
   - media query **逐字節錄**自實際產出檔（含原註解，不得自加行尾註解）。
   - `**圖 N-X　同一個檔案在手機寬度下的單欄＜主內容＞**`（X 為下一個順序圖號）+ 圖檔 pointer 行。
   - 業態需求對照段：把產出區塊對回第 2 節的需求。
   - 收尾提醒：AI 每次產出不會一模一樣，要驗收的是下一小節的檢查項目。
5. **真實產出網站**（5.2 的素材，禁 mockup）：
   - 嚴格照本章 5.1 prompt 生成單檔 `index.html` → 存 `04-build/p15/lesson-NN/index.html`。
   - 要求：語意化 HTML、CSS 同檔、圖片用**已 curl 驗證 200 的免費圖庫（Unsplash）主題圖** + `<!-- 替換為 images/xxx.jpg -->` 註解（6/13 起棄用灰色佔位圖：書中截圖要像真網站）、全部 img 有 alt、RWD 照 prompt（通常桌機三欄/平板兩欄/手機單欄）、表單欄位齊、**不得捏造評價/獎項/合作品牌**、SEO title + meta description。
   - 手機版 header 不得醜陋斷行（參考講 3：640px 以下 header 改 column）。
6. **截圖**（playwright python，截完即關 browser）：
   - 桌機 1440x900 viewport 首屏 → `04-build/p15/figures/fig-NN-0X-home-desktop.png`（X 照講 3 慣例接續本講圖號連號，例：`fig-03-02`）
   - 手機 390x844 捲到主內容區（要看得出單欄）→ `04-build/p15/figures/fig-NN-0Y-mobile-<slug>.png`（Y 為下一個連號，例：`fig-03-03`）
   - 已交付章的既有檔名（含早期 `0a`/`0b` 式）不回頭改；md pointer 行與實檔完全一致即合格。
   - 截完用 Read 工具看圖確認：無斷行災難、placeholder 標籤有渲染、確實單欄。不合格就修 HTML 重截。
7. **流程圖升級**（節制）：本章若有 1-2 條「核心」文字箭頭流程（如更新流程、生命線級別），升級為編號圖：引入句改「如圖 N-X 所示（請排版將下列步驟製成橫式流程圖）」+ 粗體圖題行。其餘箭頭流程保持原樣。圖號與截圖按出現順序統一排。交排版時隨附 `figures/FIGURE-MAP.md`（全書圖號↔圖檔名對照表），配圖以對照表與 md pointer 行為準。
8. **§6 結構**：若本章已有 6.1/6.2/6.3 小節，保持；若沒有（如講 1），照講 3 補 `### 6.1 上線檢查清單` / `### 6.2 延伸功能與實作作業` / `### 6.3 參考文獻`。
9. **參考文獻**：升級為 `### 6.3 參考文獻`（已存在則只整理格式）。格式：`來源. 標題. URL （擷取日期：2026-06-12）`。保留既有來源；可補 1-2 個本章對口的官方技術來源（白名單：MDN `Learn_web_development` 路徑、web.dev、GitHub Docs、Vercel Docs、Google Search Central、Stripe Docs（金流章）、W3C WAI（無障礙）、schema.org）。每條 URL 都要 curl 驗 200。
10. **體例小修**：中英文之間補半形空格（regex 掃）、縮寫全稱只在章內首次出現展開一次（表格內一律用縮寫）、60 字以上無逗點長句補標點。手術式，不重寫。
11. **技術地圖錨點**（限定章）：講 1 在第一次部署段落補一句「部署背後的版本控制與自動部署原理，會在第 3 講正式拆解」；講 18 在交付收束處補一句回扣第 3 講主場。其他章不加。
