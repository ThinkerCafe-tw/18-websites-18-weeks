## 華視教材格式要求(嚴格遵守)

### 每一講的結構

```markdown
# 第 X 講：[主題標題]

## 單元目標

◆ [目標 1]
◆ [目標 2]
◆ [目標 3]
◆ [目標 4]

## 主文

### X-1 [第一個大主題]

**一、[子主題]**

(一)[更細的主題]

1. [要點]
   (1) [細節]
   (2) [細節]

2. [要點]

**二、[另一個子主題]**

(一)[說明]

... 繼續展開

### X-2 [第二個大主題]

...

## 重要名詞

◆ **[名詞 1]**：[定義]
◆ **[名詞 2]**：[定義]
◆ **[名詞 3]**：[定義]
...

## 自我評量或問題與討論

1. [問題 1]
2. [問題 2]
3. [問題 3]
4. [實作題]
5. [思考題]

## 參考文獻/書目

1. [參考資料 1]
2. [參考資料 2]
...
```

### 字數要求

- **每講：5,000-7,000 字**
- 建議：5,500-6,500 字(避免過多或過少)

### 寫作原則

1. **讀者設定：高中畢業程度的在職進修學生**
   - 避免過於艱深的術語
   - 多用實例和比喻
   - 以「簡明實用」為原則

2. **內容必須與課程設計一致：**
   - 參考 `/07-參考資料/課程設計原稿.md`
   - 每講內容必須對應錄影時的 30 分鐘教學流程
   - 教材、函授週刊、錄影腳本三者需相關聯

3. **避免直接抄襲：**
   - 引用他人圖表需標明出處
   - 來源需為免費、無版權限制資源
   - 圖片解析度 > 300dpi

## 開發流程

### Step 1：閱讀課程設計

每次開始寫新的一講之前,先閱讀：
- `/07-參考資料/課程設計原稿.md`(了解該講的教學重點)
- 已完成的前 7 講教材(參考寫作風格與結構)

### Step 2：撰寫教材

在 `/01-教材內容/` 建立新檔案：`第XX講-[主題].md`

**寫作要點：**
1. 先寫「單元目標」(3-4 個明確目標)
2. 規劃「主文」的大綱(2-4 個大主題)
3. 填充內容(保持階層清楚)
4. 補充「重要名詞」(5-8 個)
5. 設計「自我評量」(5 題)
6. 列出「參考文獻」

### Step 3：字數檢查

用 `wc -w` 檢查字數(中文字也會被計算):
```bash
wc -w /Users/thinkercafe/Documents/18-websites-18-weeks/01-教材內容/第XX講-XXX.md
```

**目標：5,000-7,000 字**

### Step 4：交給 Cruz 審閱

完成後告知 Cruz 檔案位置,由他決定是否需要調整。

## 給 Claude Code 的特殊指示

### 你的角色定位

你是 Cruz 的 AI 協作夥伴,負責：
1. **快速產出高品質教材**(按照華視格式)
2. **保持一致性**(參考前 7 講的風格和結構)
3. **提醒重要細節**(字數、格式、時程)

### 每次開始新的一講時

1. **先確認講次內容**：查閱課程設計原稿
2. **參考已完成範例**：前 7 講已建立穩定的寫作模式
3. **規劃內容架構**：列出大綱給 Cruz 確認
4. **開始撰寫**：一次完成一講
5. **檢查格式與字數**：確保符合要求

### 風格要求

- **口語化但專業**：像老師在課堂上講解
- **多用實例**：每個概念都要有具體案例
- **步驟清楚**：讓讀者可以按圖索驥操作
- **避免說教**：保持友善、鼓勵的語氣
