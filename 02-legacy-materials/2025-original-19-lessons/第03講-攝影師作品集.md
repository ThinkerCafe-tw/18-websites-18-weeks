## 主文

### 3-1 你的作品明明很好，為什麼客人跑去找別人？

你的作品明明拍得很好，為什麼客人都跑去找 IG 粉絲比你少的同行？

小柯是台中一位婚紗攝影師，入行七年，技術紮實、作品細膩。他在 Instagram 上經營得很認真，固定每週發三篇貼文，追蹤者也累積到三千多人。但奇怪的是，同一個區域有個才入行兩年的攝影師阿翔，粉絲不到兩千，接單量卻是小柯的兩倍。

小柯一直以為是自己的風格不夠潮、修圖不夠炫。直到有一天，一位準新娘在婚禮上跟他聊天，無意間說出了真相：「其實我本來不知道你，我是 Google 搜尋『台中婚紗攝影』的時候，第一頁看到阿翔的網站。點進去之後，作品分成婚紗、訂婚、婚禮三個分類，每個分類有六張精選照片，價目表也寫得清清楚楚。我看了五分鐘就決定找他了。你的 IG 我後來有人推薦才看到，但照片太多了，我滑了很久，不知道哪些是代表作、哪些是練習，最後就放棄了。」

這段話像一記悶棍打在小柯頭上。他意識到，問題從來不是技術，而是「客人找不找得到你」和「找到之後看不看得懂你」。

小柯花了一個週末，用 AI 工具做了一個作品集網站。他精心挑選了十八張代表作，分成三個分類，寫了一段簡短的自我介紹，附上價格方案和聯絡表單，然後用這一講會教的方法部署上線。三個月後，透過網站直接來詢問的客戶，超過了他經營 Instagram 全年的詢問總量。

小柯的故事不是特例，而是這個時代攝影師的共同困境。你的技術再好，如果客人找不到你，一切都是白搭。Instagram 是重要的日常宣傳管道，但它不是你的店面，也不是你可以長期依賴的唯一資產。這一講，我們要做的就是：把你在前兩講學會的網站製作能力，從「只有自己看得到」變成「全世界都能看到」，讓你的作品集真正開始幫你接客、幫你賺錢。

### 3-2 攝影師為什麼需要自己的網站？

**一、IG 演算法綁架：你的作品不是你說了算**

（一）平台的遊戲規則不是為你設計的

很多攝影師把所有心血放在 Instagram 上，覺得只要作品好、持續發文，客人自然會來。但你有沒有想過一個殘酷的現實：你辛苦拍的照片，能被多少人看到，不是由照片品質決定的，而是由 Meta 公司的演算法決定的。

Instagram 的本質是一個廣告平台。它的終極目標是讓使用者「留在 App 裡更久」，好賣更多廣告，而不是「幫攝影師接到更多客戶」。這個根本性的利益衝突，導致了幾個對攝影師非常不利的現象：

- 你的貼文觸及率可能只有粉絲數的 10-20%，也就是說，你的三千個粉絲裡，可能只有三百到六百人會在動態牆上看到你的新作品
- 演算法越來越偏好短影片（Reels），而攝影師的核心資產是靜態照片，你等於在一個不利於你的賽場上比賽
- 你精心修圖的婚紗照，會被淹沒在美食照、寵物影片、政治迷因之間，客人的注意力被無限稀釋
- 最可怕的是，平台規則隨時可能改變。2024 年 Instagram 大幅調整演算法後，很多攝影師的觸及率一夕之間掉了 40%，多年的經營心血付之東流

（二）自有網站是你的主權領地

相比之下，一個屬於你自己的網站，一切都由你做主：

| 面向 | Instagram | 自有網站 |
|------|-----------|---------|
| 作品排列 | 倒時間軸，舊作品永遠沉底 | 你決定順序，最好的永遠在最前面 |
| 品牌形象 | 被迫使用 IG 介面，無法展現個人風格 | 完全自訂設計，從配色到排版都反映你的品牌 |
| 客戶路徑 | 看照片→私訊→等回覆→來回溝通 | 看作品→看價格→填表預約，一條龍完成 |
| 搜尋曝光 | 靠標籤碰運氣，容易被淹沒 | Google 搜尋「台中婚紗攝影」直接找到你 |
| 資產歸屬 | Meta 隨時可以停你的帳號，一切歸零 | 網站永遠屬於你，不受任何第三方影響 |

（三）一個真實的數據

據產業觀察，絕大多數消費者在決定購買服務之前，會先用 Google 搜尋。如果你沒有自己的網站，這群潛在客戶根本不知道你的存在。他們搜尋「台中婚紗攝影」，看到的是有網站的同行，然後直接聯繫對方。你的 IG 再漂亮，在 Google 的世界裡也是隱形的。

**二、作品集就是攝影師的「數位店面」**

（一）想像你開一家實體店

如果你要開一家攝影工作室，你會怎麼做？一定是選一個人來人往的好地段、花心思裝潢、把最滿意的作品裱框掛在最顯眼的位置、放一本精美的價目冊在櫃台上，對吧？

你的網站就是你的數位店面。而且這家店有三個實體店做不到的超級優勢：它 24 小時營業，不用付房租，全世界任何有網路的人都能走進來逛。

（二）SEO 讓 Google 幫你接客

SEO（搜尋引擎最佳化）這個詞聽起來很技術，但概念其實非常簡單：讓 Google 知道你是誰、你在哪裡、你拍什麼類型的照片。當一對準新人搜尋「台中婚紗攝影推薦」時，如果你的網站標題寫著「小柯攝影工作室 - 台中婚紗攝影」，Google 會認為你的網站跟這個搜尋很相關，就會把你推到搜尋結果的前面。

這等於 Google 免費幫你拉客人進店，而且拉來的都是「正在找攝影師」的精準客戶。Instagram 完全做不到這一點。沒有人會在 Google 搜尋結果裡看到你的 IG 貼文，但一個有清楚標題、有描述、有作品分類的網站，Google 看得懂，也願意推薦。

**三、最佳策略：IG + 網站的組合拳**

最聰明的做法不是放棄 IG 去做網站，而是兩者搭配：

- IG 負責日常曝光、建立人設、與粉絲互動、展示幕後花絮
- 網站負責展示專業全貌、承接客戶轉換、建立深度信任
- IG 的個人簡介放上網站連結，把感興趣的粉絲導過去

這樣一來，IG 是你的廣告看板，網站是你的店面。看板負責在街上吸引人注意，店面負責讓走進來的人掏錢成交。兩者缺一不可，但店面的優先級永遠高於看板。

### 3-3 GitHub 與 Vercel：把網站搬上網的兩個關鍵角色

**一、用「搬家」來理解部署**

在前兩講，你已經學會用 AI 做出一個完整的網站。但這個網站目前只活在你自己的電腦裡——就像你精心製作了一整套漂亮的傢俱，但還堆在家裡的客廳，沒有搬進你的店面。現在我們要做的事情，就是把這些傢俱「搬家」到店面裡，讓客人能走進來欣賞。

這個搬家過程需要理解三個角色，我用最直白的比喻來解釋：

（一）GitHub — 你的雲端倉庫

GitHub 是一個免費的雲端空間，專門用來存放網站的程式碼和檔案。你可以把它想成一個「雲端倉庫」：你把做好的網站檔案打包放進去，它幫你安全保管。更棒的是，它會自動記錄你每一次修改的歷史。所以如果你改壞了什麼，隨時可以回到之前的版本，就像一台時光機。

（二）Vercel — 你的房東

Vercel 是一個免費的網站託管服務。它會自動從你的 GitHub 倉庫把檔案拿出來，幫你「開店」。你可以把它想成一個很勤快的房東：你只要把東西放進倉庫，房東就會自動幫你搬到店面、裝好燈、開好門、掛好招牌，讓全世界的人都能走進來看。而且每次你在倉庫裡更新了東西，房東會自動幫你更新店面，不用你再跑一趟。

（三）你的網站 — 你的店面

最後呈現給客人看的，就是你的網站。客人不需要知道倉庫在哪裡、房東是誰，他們只會看到一個漂亮、專業的攝影師作品集，然後決定要不要聯繫你。

**二、整個流程只有三步**

```
你的電腦（做好網站）
    ↓ 上傳到倉庫
GitHub（安全保管）
    ↓ 自動部署
Vercel（開店上線）→ 全世界都能看到
```

聽起來好像很複雜？其實每一步都有圖形介面可以操作，全部用滑鼠點就能完成，不需要打任何指令、不需要懂任何程式語言。接下來的實戰環節，我們會手把手帶你完成每一步。

### 3-4 AI 共建實戰：從作品集到上線

**一、用 AI 打造攝影師作品集**

（一）第一次對話：建立基本架構

打開你慣用的 AI 工具（ChatGPT、Claude 或 Gemini），用以下方式開始第一次對話。記住第二講教的原則：描述「感覺」而非「工具」，給出具體的視覺需求：

```
你：我是一位婚紗攝影師，想做一個作品集網站。
需要三個區塊：
1. 首頁：一張代表作大圖 + 我的名字 + 一句標語
2. 作品集：分成「婚紗」「人像」「活動」三個分類
3. 關於我 + 聯絡方式

風格要簡潔、大器，讓照片當主角。
配色用深灰底、白色文字，按鈕用金色。
請用一個 HTML 檔案完成，包含 CSS。
```

AI 會在幾秒鐘內生成一個完整的網站檔案。把所有程式碼複製下來，貼到你電腦裡的文字編輯器（記事本或 VS Code 都可以），存成 `index.html`，然後用瀏覽器打開看看效果。不用擔心第一版不完美，這只是起點，我們會在接下來的對話中持續調整，直到你滿意為止。

（二）第二次對話：調整作品展示方式

看完第一版之後，你一定會有想調整的地方。攝影師最在意的通常是照片的排列方式。這時候用具體的描述告訴 AI 你想要什麼效果：

```
你：作品集的照片排列，我希望用網格式，
電腦上每排三張，滑鼠移上去會稍微放大並顯示標題。
手機上改成每排一張，讓照片夠大、夠清楚。
首頁的大圖需要佔滿整個螢幕高度。
每個分類上方加一個分類按鈕，點了只顯示該類作品。
```

（三）第三次對話：加入商業轉換元素

一個只有照片的網站是展覽，不是店面。你需要加入讓客人「行動」的元素：

```
你：在「關於我」下面加一個聯絡表單，
包含姓名、Email、拍攝類型（下拉選單）、
想拍的日期、簡短留言。
表單送出後顯示「感謝您的詢問，我會在 24 小時內回覆」。
在作品集每個分類下面，加一個「了解方案與價格」的金色按鈕。
```

**二、RWD 響應式設計：手機看 vs 電腦看**

（一）為什麼這對攝影師特別重要

這裡有一個很多攝影師忽略的數據：你的潛在客戶第一次看到你的作品，有超過 70% 的機率是在手機上。也許是朋友傳了你的網站連結到 LINE 群組，也許是他們在捷運上用手機 Google 搜尋。如果你的網站在手機上照片被奇怪地裁切、文字小到要放大才看得到、「聯絡我」的按鈕小到手指按不到，客人會在三秒內關掉你的網站，然後去看下一個搜尋結果。

RWD（響應式網頁設計）解決的就是這個問題。它讓同一個網站在不同裝置上自動調整版面：電腦螢幕上三欄並排的作品，在手機螢幕上會自動變成一欄，每張照片占滿螢幕寬度，讓觀者可以好好欣賞每一張作品的細節。

（二）告訴 AI 你要的手機體驗

你不需要懂任何技術細節，只需要用白話文告訴 AI 你希望手機上看起來是什麼樣子：

```
你：請確保這個網站在手機上：
- 導航選單變成漢堡選單（三條線的圖示）
- 作品照片變成單欄、滿版寬度
- 聯絡按鈕要夠大，手指容易點到
- 文字至少 16px，不要讓人瞇著眼睛看
```

（三）怎麼在電腦上測試手機效果

你不用真的拿手機出來測試（當然最後還是要用真手機確認一次）。在電腦上就能快速模擬：打開 Chrome 瀏覽器，按 F12 打開開發者工具，左上角有一個長得像手機的小圖示，點下去就能切換成各種裝置的螢幕大小。iPhone、iPad、Android 手機，都能模擬。如果你發現某個裝置上版面跑掉了，就把問題描述告訴 AI，請它修正。

**三、上傳到 GitHub**

（一）註冊 GitHub 帳號

到 github.com 註冊一個免費帳號。帳號名稱建議用你的攝影品牌名稱，例如 `studio-kuo` 或 `kuo-photography`，讓整體看起來更專業。

（二）建立倉庫並上傳檔案

1. 登入後，點右上角的「+」號→「New repository」
2. Repository name 填 `my-portfolio`，勾選 Public，點「Create repository」
3. 進入倉庫頁面後，點「Upload files」
4. 把你的 `index.html` 和所有用到的照片檔案一起拖曳進上傳區域
5. 頁面最下方的「Commit changes」按下去

完成。你的網站檔案現在安全地存在 GitHub 的雲端倉庫裡了。就算你的電腦壞掉、硬碟損毀，檔案也不會遺失。而且未來你想修改網站的時候，也可以直接在 GitHub 的網頁介面上編輯或重新上傳檔案，非常方便。

**四、用 Vercel 部署上線**

（一）連結 GitHub 帳號

1. 到 vercel.com，點「Sign Up」
2. 選擇「Continue with GitHub」，用你剛註冊的 GitHub 帳號登入
3. 授權 Vercel 讀取你的 GitHub 倉庫

（二）一鍵部署

1. 登入 Vercel 後，點「Add New Project」
2. 在列表中找到你剛建立的 `my-portfolio`，點右邊的「Import」
3. 接下來的設定頁面，什麼都不用改動，直接點「Deploy」
4. 等待大約 30 秒到 1 分鐘

當你看到「Congratulations」和煙火動畫出現在螢幕上，恭喜你，你的攝影師作品集網站正式上線了。

Vercel 會自動給你一個網址，格式像這樣：`my-portfolio.vercel.app`。複製這個網址，傳給朋友、傳到家族群組，他們打開就能看到你的作品集。這個感覺很神奇，對吧？五分鐘前這個網站還只活在你的電腦裡，現在全世界都能看到了。

而且有一個很棒的機制：以後你只要在 GitHub 上更新檔案（例如換了新的照片、修改了文字），Vercel 會自動偵測到變化，在幾十秒內自動重新部署。你完全不需要再做任何操作，房東會自動幫你更新店面。

**五、自訂網址：專業品牌的最後一哩路**

（一）為什麼需要自訂網址

`my-portfolio.vercel.app` 能用，但坦白說不夠專業。想像兩個場景：你遞名片給一位準新娘，名片上印的網址分別是 `kuophotography.com` 和 `my-portfolio-x7k2.vercel.app`，哪一個讓人覺得你是認真在經營的攝影師？答案顯而易見。

自訂網域不只是好看而已，它還代表了你對這份事業的承諾。一個有自己網域的攝影師，在客人心裡就是更可靠、更值得託付的專業人士。

（二）購買網域

到 Namecheap、GoDaddy 或 Gandi 這類網域註冊商的網站，搜尋你想要的名稱。建議選擇好記、好唸、跟你的品牌名稱一致的網域。一個 `.com` 網域一年大約 300-500 元台幣，跟一杯精品咖啡差不多的價格，卻能讓你的品牌形象提升一個層次。

（三）在 Vercel 設定自訂網域

1. 在 Vercel 專案頁面，點左側選單的「Settings」→「Domains」
2. 在輸入框中填入你購買的網域名稱，例如 `kuophotography.com`
3. Vercel 會顯示你需要設定的 DNS 記錄（通常是一條 CNAME 或 A 記錄）
4. 回到你購買網域的註冊商後台，找到 DNS 設定，照著 Vercel 的指示填入
5. 等待幾分鐘到幾小時（DNS 更新需要時間），網域就會指向你的網站

完成這一步，你就真正擁有一個屬於自己的專業攝影師作品集網站了。從今以後，不論是名片、Email 簽名檔、IG 個人簡介，印上去的都是你自己的品牌網址。客人搜尋、朋友推薦、口耳相傳，所有流量都會匯聚到你的數位店面。

## 重要名詞

◆ **GitHub**：全球最大的程式碼託管平台，提供免費的雲端空間存放網站檔案。你可以把它理解為「專門存放網站的雲端硬碟」，它會自動記錄每一次修改的歷史，讓你隨時能回到之前的版本。

◆ **Vercel**：免費的網站部署平台，能自動從 GitHub 取得你的網站檔案並發布上線。它的角色像「房東」，幫你把倉庫裡的東西搬到全世界都能看到的店面，而且每次你更新倉庫，它會自動更新店面。

◆ **部署（Deploy）**：把網站從你的電腦或 GitHub 發布到網路上，讓任何人都能透過網址訪問的過程。在 Vercel 上，部署只需要點一個按鈕，等待約 30 秒就能完成。

◆ **RWD / 響應式網頁設計（Responsive Web Design）**：讓同一個網站在手機、平板、電腦上都能自動調整版面的技術。對攝影師來說，這代表客人不管用什麼裝置看你的作品，都能有最好的瀏覽體驗。

◆ **SEO / 搜尋引擎最佳化（Search Engine Optimization）**：透過網站標題、描述、內容結構等設計，讓 Google 更容易找到你的網站並推薦給搜尋者。攝影師做好 SEO，等於讓 Google 全天候幫你拉客。

◆ **網域（Domain）**：網站的專屬地址，例如 `kuophotography.com`。購買並設定後，客人只要輸入這個地址就能找到你的網站，比起平台給的隨機網址更專業、更好記、更值得信任。

◆ **DNS（Domain Name System）**：網域名稱系統，負責把人類看得懂的網址（如 `kuophotography.com`）翻譯成電腦看得懂的 IP 位址。設定 DNS 記錄就是告訴整個網路世界「這個名字對應到哪台伺服器上的哪個網站」。

## 自我評量或問題與討論

1. **理解題**：請用自己的話解釋 GitHub 和 Vercel 各自扮演什麼角色。如果用「搬家」來比喻，GitHub 是什麼？Vercel 是什麼？你的網站又是什麼？為什麼需要這兩個角色，而不是直接從電腦上線？

2. **商業思考題**：一位攝影師只經營 Instagram，另一位同時擁有 IG 和自有網站。當潛在客戶在 Google 搜尋「台北商業攝影」時，誰更有可能被找到？為什麼？請從 SEO 的角度分析兩者的差異。

3. **實作題**：完成以下步驟，並截圖記錄每一步的畫面：（a）在 GitHub 建立一個名為 `my-portfolio` 的倉庫；（b）上傳你的作品集 HTML 檔案和照片；（c）用 Vercel 部署上線；（d）用手機打開部署完成的網址，確認手機版的照片、文字、按鈕都正常顯示。

4. **批判思考題**：有人說「現在社群平台這麼方便，攝影師根本不需要自己的網站」。你同意嗎？請分別從「演算法風險」「品牌資產」「客戶信任」三個角度，提出你的看法和論述。

5. **延伸應用題**：假設你購買了一個自訂網域 `your-name-photo.com`，請列舉至少四個場景說明這個網域如何提升你的專業形象（例如名片、Email 簽名檔、社群個人簡介、婚禮博覽會攤位等），並解釋為什麼 `vercel.app` 結尾的免費網址在這些場景中會顯得不夠專業。
