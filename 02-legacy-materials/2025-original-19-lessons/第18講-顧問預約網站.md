# 第十八講：顧問預約網站 — 金流整合與商業閉環

## 單元目標

◆ 學會建立顧問專業形象頁面，讓訪客在三十秒內理解你的專業與收費方式  
◆ 掌握收費諮詢預約系統的設計邏輯，包含免費初談與付費深談的分層機制  
◆ 理解金流串接的核心概念——不用自己寫程式，理解綠界和藍新的跳轉流程就夠  
◆ 建立從「預約→付款→服務→發票→回饋」的完整商業閉環，讓網站真正產生收入

## 主文

### 18-1 破冰：你的專業值多少？

你的專業值多少？如果連收費都不敢放在網站上，客戶怎麼會認為你值得付費？

這個問題聽起來尖銳，但它戳中了大多數顧問最大的心理障礙。在前面十七講中，我們做過咖啡廳、健身房、診所、民宿等各種網站。這些行業有一個共同特點：顧客付費後會拿到「看得到的東西」——一杯咖啡、一堂課、一個房間。但顧問不一樣。

顧問賣的是經驗、判斷力和解決問題的能力。這些東西看不到、摸不到、拍不了照。客戶在付費前，需要比任何其他服務都更多的「信任線索」。而信任的起點，往往就是你敢不敢把價格公開地放在網站上。

很多新手顧問犯的第一個錯誤就是「價格私訊」——網站上寫著「費用請私訊洽詢」。這句話翻譯成客戶心裡的聲音就是：「他大概很貴，而且可能會看人開價。」結果是大量潛在客戶直接離開，連私訊都懶得發。

反直覺的真相是：公開價格反而篩選出真正的客戶。願意付你開出的價格的人才會預約，不願意的人不會浪費你的時間。這就是這一講要建立的思維——金流不是技術問題，而是商業信任的具體展現。

而且，這一講是進階收官講。前面十七講學過的所有概念——形象設計、預約系統、表單邏輯、一對一時段管理、隱私保護——全部會在這裡匯合。顧問預約網站就是前面所有技能的期末考。

### 18-2 商業洞察：免費初談到付費深談的轉換設計

**一、時間就是商品：顧問預約的特殊邏輯**

對顧問來說，時間就是唯一的商品。不像咖啡廳可以同時服務二十個客人，顧問在一個時段只能服務一個人。每一個被爽約的時段，都是直接的收入損失——而且無法「補貨」。

這讓預約系統的設計邏輯跟前面幾講完全不同。診所預約如果有人爽約，下一位候診的病人可以遞補。但顧問的一小時諮詢，如果客戶沒出現，那一小時就蒸發了。

根據業界經驗：免費預約的爽約率通常較高，可能達到四分之一到三分之一；而付費預約的爽約率則大幅降低，僅剩個位數百分比。這意味著付費機制可以顯著提升服務交付率（以下為教學假設情境，實際數據因行業與設計而異）。

心理學上有一個解釋：一致性原則。一旦人做了承諾（付款），就會傾向符合那個承諾（出席）。客戶的「付出」越早發生，他對服務的期待和專注度越高。

從另一個角度看，付費預約還有一個隱性好處：它幫你篩選掉不認真的客戶。免費預約的門檻太低，有些人只是隨手點一下，並沒有真正的諮詢需求。這些人佔了你的時段卻不出現，浪費的不只是你的時間，還有其他真正需要幫助的客戶的機會。付費機制確保坐在你面前的每一個人，都是帶著真實需求和認真態度來的。

**二、分層設計：免費初談 + 付費深談**

所以最優的顧問預約設計是分兩層。

第一層是免費初談，十五到三十分鐘，零成本，目的是降低嘗試門檻。客戶在這個階段只需要填最基本的資訊——姓名、聯絡方式、簡述需求。免費初談不是白工，而是「用三十分鐘換一個付費客戶的高確定性機會」。

第二層是付費深談，六十到九十分鐘，兩千到一萬元不等，需要先付款才能預約。這一層才是真正的價值交付。客戶在這個階段需要填寫完整資訊，因為付了錢代表他是認真的。

這個分層策略的商業邏輯很清楚：免費初談是漏斗的入口，付費深談是漏斗的出口。中間的轉換率取決於你在初談中展現的專業度——這不是網站能決定的，是你的本事。但網站可以做的是：讓這個漏斗的入口夠寬（免費初談門檻低）、出口夠順暢（付款流程不卡住）。

**三、金流不是寫程式——理解商業閉環**

很多人聽到「金流串接」就害怕了，以為要學複雜的加密算法或支付協議。事實上，對於使用 AI 工具建站的你來說，金流最核心的不是技術，而是理解金錢流動和商業信任。

完整的商業閉環是一個自我強化的飛輪：曝光（搜尋、社群、推薦）→ 訪客看到專業形象頁（產生初步信任）→ 預約免費初談（低門檻嘗試）→ 初談中了解需求 → 客戶決定付費深談 → 線上付款確認 → 進行諮詢 → 開立發票（法規合規）→ 邀請留下評價 → 評價顯示在網站 → 新訪客看到評價 → 提升信任 → 更多預約。

每一個滿意的客戶都成為下一批訪客的信任信號。這就是為什麼網站不只是「名片」，而是生意的引擎。

### 18-3 核心概念：金流串接的本質

金流串接的本質是「你的網站→金流商→銀行」之間的信息流。理解這個流程，比寫任何程式碼都重要。

當客戶在你的網站點擊「付款」，你的網站發送訂單資訊（金額、品名、訂單編號）給金流商。金流商接手付款處理——客戶在金流商的加密頁面輸入卡號，不在你的網站。銀行確認扣款後，金流商通知你的網站「付款成功」。你的網站標記預約為已確認，並發送確認信給客戶。

關鍵安心點：客戶的信用卡號永遠不會經過你的網站。整個過程中，敏感的金融資訊只在金流商的加密環境中處理。這叫「跳轉型付款」——最安全、最簡單、初學者最適合的方式。

台灣最常用的兩家金流服務商：綠界 ECPay（市佔率最高，個人戶可申請，手續費約百分之二點七五）和藍新 NewebPay（市佔率第二，需公司登記，手續費約百分之二點八）。兩家的核心邏輯完全一樣，學會一家就能通用。初學者建議先用綠界的測試環境練習。

所有金流商都提供測試環境（Sandbox），用假信用卡號完整跑一遍流程，不會真的扣款。這是你練習的安全空間。

除了跳轉型，金流還有兩種進階模式值得了解。內嵌型付款是在你自己的網頁上放信用卡輸入框，客戶不用跳到外部頁面，品牌體驗更統一，但技術門檻和安全責任都更高。行動錢包（Apple Pay、Google Pay）則是最新的趨勢——客戶用指紋或臉部辨識一秒完成付款，體驗最好但整合最複雜。初學者先學跳轉型就夠了，概念通了之後升級到內嵌型只是換一段程式碼，底層邏輯完全一樣。

還有一個重要的概念是「付款完成通知」。當客戶在金流商的頁面完成付款後，金流商會透過 Webhook 機制主動通知你的網站。Webhook 就像是金流商派一個信差，跑到你的網站門口敲門說：「這筆錢收到了，訂單編號是某某某。」你的網站收到這個通知後，就自動標記預約為「已付款」、發送確認信給客戶。整個過程不需要人工介入。

### 18-4 AI 實戰：建立收費顧問預約網站

**第一輪：專業形象頁面**

顧問網站的首頁要在三十秒內完成三件事：讓訪客知道你是誰、你能幫什麼忙、怎麼開始。

```
提示詞範例：

請幫我設計一個管理顧問的個人網站首頁，包含五個區塊：

1. Hero 區：
   - 標題：「讓你的團隊從混亂走向高效」
   - 副標題：「10 年企業管理經驗，協助超過 50 家中小企業優化營運流程」
   - 兩個按鈕：「免費初談 15 分鐘」（主按鈕）和「了解服務方案」（次按鈕）

2. 服務方案區（三欄對比）：
   - 免費初談：15 分鐘、線上視訊、了解需求、零風險
   - 策略諮詢：60 分鐘、NT$3,000、深度診斷一個具體問題
   - 陪跑顧問：月制 NT$15,000、每週一次定期會議、持續追蹤

3. 客戶見證區：三則推薦語（用引號卡片呈現）

4. 關於我：簡短經歷、專業背景、照片預留區

5. 預約行動區：大型 CTA 按鈕「預約免費初談」

配色：深藍 #1B365D + 金色 #C9A84C + 白色
風格：專業、值得信賴、有份量感
```

注意服務方案區的三欄設計。這是經典的定價心理學——當你提供三個選項，大多數人會選中間那個。這個現象叫「錨定效應」：最貴的選項不是為了賣出去，而是讓中間選項看起來「很合理」。「免費初談」降低門檻，「策略諮詢」是主力產品，「陪跑顧問」拉高天花板讓中間選項顯得物超所值。

**第二輪：分層預約表單**

免費初談和付費深談的表單設計完全不同。

```
提示詞範例：

請幫我設計兩個預約表單，放在同一個頁面上用 Tab 切換：

Tab 1：免費初談預約
- 欄位：姓名、Email、手機（三選一即可）、希望時段（下拉選單）
- 簡述需求（100字內，選填）
- 按鈕文字：「預約免費初談」
- 備註：「初談為 15 分鐘線上視訊，完全免費，不會有任何推銷壓力」

Tab 2：付費策略諮詢預約
- 欄位：姓名、Email、手機、公司名稱（選填）、希望時段
- 詳述需求（500字內，必填——付費客戶需要認真描述問題）
- 費用說明：「策略諮詢 60 分鐘 / NT$3,000」
- 按鈕文字：「確認預約並前往付款」
- 取消政策：「48 小時前取消可全額退費」

時段選擇要自動排除已被預約的時段（灰色不可選）
```

這裡的設計細節很重要：免費初談的備註「不會有任何推銷壓力」直接化解了客戶最大的疑慮。付費諮詢要求「詳述需求」是合理的——花三千元的人有權利被認真對待，而認真描述問題的過程本身也幫助客戶釐清自己到底需要什麼。

**第三輪：金流概念圖與付款流程**

你不需要真的串接金流 API（那是工程師的活），但你需要理解流程並設計出對應的頁面。

```
提示詞範例：

請幫我設計金流付款的三個頁面：

頁面 1：付款前確認頁
- 顯示預約摘要：日期、時間、服務項目、金額
- 取消政策再次提醒
- 「確認並前往付款」按鈕
- 說明：「點擊後將跳轉至綠界安全付款頁面，您的信用卡資訊不會經過本網站」

頁面 2：付款成功頁
- 大勾勾圖示 + 「付款成功！」
- 訂單編號（格式：CONSULT-20260409-001）
- 預約摘要
- 「確認信已寄至您的信箱」
- 「加入行事曆」按鈕

頁面 3：付款失敗頁
- 提示可能原因（信用卡額度不足、網路問題等）
- 「重新付款」按鈕
- 「改用其他付款方式」選項
- 客服聯絡方式
```

訂單編號的設計看起來是小事，但它是信任的根基。客戶可以說「我付了 CONSULT-20260409-001」，你可以查系統確認。這個編號就是雙方信任的具體載體。

**第四輪：發票與退費機制**

在台灣收錢就要開發票或收據，這不是可選項目，而是法律義務。

```
提示詞範例：

請幫我設計發票和退費相關的頁面元素：

1. 發票選項（在付款確認頁）：
   - 個人電子發票（存載具或 Email）
   - 公司三聯式發票（需填統一編號和公司名稱）
   - 捐贈發票

2. 退費政策頁面：
   - 標題：「我們的退費承諾」
   - 48 小時前取消：全額退費，3-5 個工作天入帳
   - 24-48 小時前取消：退 50%
   - 24 小時內取消：恕不退費，但可改期一次
   - 顧問方取消：全額退費 + 下次諮詢 9 折
   - 底部：「有任何問題，請直接聯繫我們，我們會盡力協助」

3. FAQ 區塊（常見問題）：
   - 「付款後多久會收到確認？」→ 即時
   - 「可以用什麼方式付款？」→ 信用卡、ATM、超商代碼
   - 「發票什麼時候開？」→ 付款完成後 48 小時內
   - 「如果諮詢後覺得沒幫助可以退費嗎？」→ 誠實回答
```

退費政策的設計是一門藝術。「不開心可以退」這句話的潛台詞其實是「我很自信」。清楚的退費規則不是在示弱，而是在說「我不怕被比較」——這反而增加信任。

另外要特別注意發票的處理。在台灣，有公司登記的顧問必須開統一發票，個人身份的顧問開收據即可，但年收超過一百萬就必須辦理營業登記。初期建議用 Excel 記帳加上手動開收據，月營收穩定後再升級到電子發票系統。綠界和藍新都有提供發票代開的加值服務，額外收費但最省事。無論用哪種方式，保存所有交易紀錄和通信紀錄都是預防糾紛的關鍵。

### 18-5 成果驗收：前十七講的期末考

**一、完整網站架構檢查**

經過四輪 AI 實戰，你的顧問預約網站應該包含以下頁面：首頁（形象 + 服務方案 + 見證 + CTA）、預約頁面（雙 Tab 免費初談和付費諮詢）、付款確認頁、付款結果頁（成功和失敗）、退費政策頁、FAQ 頁、關於我頁面。

站在更高的角度回顧：這個網站動用了你在前面十七講學到的幾乎所有技能。首頁設計來自第一講的著陸頁概念。預約系統來自第四講的診所預約。一對一時段管理來自整個中期課程。隱私保護意識來自上一講的諮商預約。表單設計來自多講的累積。唯一新的元素就是金流——而你現在已經理解了它的本質。

**二、商業心理學的最終整合**

為什麼有些顧問網站能把百分之十的訪客轉化為付費客戶，而有些只有百分之一？差別不在「好看」，而在「心理設計」。

分層降低決策成本：訪客登陸，三秒內看到「你能幫我什麼」，十秒內看到免費初談按鈕，點擊後填簡單表單。每一層都減少認知負擔。

社會證明的威力：一句匿名客戶的推薦（例如「他在三個月內幫我增加百分之兩百客流」）比十句自我介紹更有說服力。

金流即承諾：付了錢的客戶，爽約可能性降低百分之八十。這就是為什麼「先付款才預約」比「預約後付款」的成功率高五到十倍。

這些都不是技術問題，而是理解人性。當你用心理學設計預約流程時，轉換率自然提升。

**三、接案報價定位**

顧問預約網站因為涉及金流整合，在市場上的定位比一般形象站高一個檔次。建議報價區間是五萬到十萬元，取決於金流是否真正串接（還是只做概念頁面）、是否含發票自動化、是否含後台管理系統。

你的報價話術：「這個網站不只是展示你的專業，它會幫你收錢、管理預約、自動發確認信、追蹤客戶回饋。它不是成本，它是你最勤勞的業務員——二十四小時上班，不請假，不抱怨。」

**四、本講的批判性反思**

最後一個提醒：金流很方便，但不要忘記最基本的事——你的服務本身要好。再漂亮的網站、再順暢的付款流程，如果客戶付了三千元卻覺得浪費時間，那一則負面評價會抵消你所有的設計努力。

金流系統讓收錢變容易，但「值不值得收這個錢」是你自己要回答的問題。網站能做的，是讓這個交易過程盡可能透明、公平、有尊嚴。剩下的，是你的專業。

回顧這一講，你學到的不只是「怎麼在網站上收錢」。你學到的是一套完整的商業思維：如何用分層設計降低客戶的嘗試門檻，如何用金流機制提升服務交付率，如何用發票和退費政策建立法規合規性，如何用商業閉環讓每一個客戶都成為下一個客戶的信任來源。這些思維，比任何程式碼都值錢。

帶著這些完整的思維走進最後一講的總複習，你會發現十八週學到的一切，都在指向同一個方向——用技術解決人的問題，用設計建立人與人之間的信任。

## 重要名詞

◆ **金流（Payment Gateway）**：線上交易的資金流動和驗證機制，連接你的網站、金流服務商和銀行。

◆ **跳轉型付款（Redirect Payment）**：客戶被導向金流商的安全頁面輸入卡號，你的網站不接觸敏感金融資訊，是初學者最適合的方式。

◆ **Webhook**：金流商付款完成後主動通知你的網站的機制，觸發後續流程如發送確認信。

◆ **測試環境（Sandbox）**：用假卡號練習完整付款流程的安全環境，不會真的扣款。

◆ **商業閉環**：從訪客到預約、付款、服務、發票、評價、再到新訪客的自我強化循環。

◆ **免費初談（Free Consultation）**：降低客戶嘗試門檻的行銷策略，用短時間零成本的接觸換取付費客戶的高確定性機會。

◆ **電子發票**：透過財政部整合平台開立的數位發票，可自動化處理。

◆ **一致性原則（Commitment and Consistency）**：心理學原理，人一旦做出承諾（如付款），會傾向符合該承諾（如出席）。
